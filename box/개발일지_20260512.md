## 1. Sentry 에러 2건 핫픽스 (커밋 `599da32`)

아침에 메일 두 통.

### DOSIGONGZON-1: `Could not find the table 'public.push_alert_log'`
`cron/storage-diet/push_alert_log` scope에서 발생. `push_alert_log` 테이블이 Supabase 프로덕션에 마이그레이션 안 된 상태. SQL은 `box/supabase_push_alert_log_migration.sql`에 있었는데 실행이 누락됨. health-alert-push도 같은 테이블 쓰는데 그쪽은 try-catch + reportError 없어서 조용히 실패 중이었던 듯 — Sentry 신호도 없이 dedup이 작동 안 하던 위험 상태.

**조치:**
- `storage-diet/route.ts`에서 PGRST205 (schema cache miss) 코드는 조용히 skip — SQL의 `to_regclass` 가드와 동일 의도
- 사용자에게 SQL Editor 실행 안내 → 실행 완료 확인

### DOSIGONGZON-2: `SyntaxError: Unexpected token '<', "<!DOCTYPE..."`
`cron/sync-pharmacies`. LOCALDATA 공공 API가 200 OK + HTML(점검·에러 페이지)을 리턴해서 `res.json()`이 폭발.

**조치:** Content-Type이 `json` 포함 안 하면 graceful skip + 다음 실행 재시도.

## 4. 동적 OG 이미지 3종 (커밋 `f756a6a`)

기존엔 `/cats/[id]`, `/community/[id]`, `/areas/[slug]`만 동적 OG가 있었음. tips·news·shorts는 단순 thumbnail URL만 써서 카톡 공유 시 미리보기가 빈약. 3개 일괄 신설.

### 디자인 톤
cats OG를 베이스로 통일:
- `/tips/[slug]` — 썸네일 좌측 + 제목·태그·💡 브랜드 우측
- `/news/[id]` — 이미지/아이콘 + 헤드라인 + 뱃지(공지/행사/정책/구조/뉴스) + 출처명
- `/shorts/[id]` — 영상 썸네일 풀스크린 + ▶ 재생 아이콘 + 제목 + 조회수·좋아요

### 함정 — 기존 metadata가 동적 OG를 무시
tips/news/shorts page.tsx의 `metadata.openGraph.images`에 `thumbnail_url` 또는 사이트 기본 OG가 명시되어 있어서 자동 적용이 안 됨. images를 동적 endpoint로 가리키게 수정:
```ts
images: [{ url: `${SITE_URL}/tips/${tip.slug}/opengraph-image`, ... }]
```

## 9. 한국 법규 컴플라이언스 4종 (커밋 `22fb41d`)

전체 약관·법적 측면 검토 요청 받고 Explore subagent로 광범위 스캔. 결과 **8.5/10 매우 양호** — PIPA 11조까지 빠짐없이, 위탁처리자 9곳 명시, 위치 처리는 본인 좌표 서버 저장 안 함 + 비로그인 70m 퍼징, 마케팅 푸시 옵트인, 신고·차단 시스템까지 다 갖춰져 있었음.

다만 한국 법규 해석상 4건 보완 필요해서 5/20 회원 모집 전 핫픽스로 일괄 적용.

### (1) 쿠키 동의 배너 — 정보통신망법 §22-2
`ConsentManager.tsx` 신설. 첫 방문 시 하단 배너 + 동의/거부 두 버튼. **동의 시에만 Vercel Analytics·SpeedInsights 로드** — 이전엔 무조건 활성이었음. layout.tsx에서 Analytics/SpeedInsights 직접 import 제거하고 ConsentManager 안으로 옮김. localStorage `dosigongzon_cookie_consent`에 영구 저장.

### (2) 가입 마케팅 동의 분리 — 정보통신망법 §22
signup 페이지에 [선택] 마케팅·이벤트 알림 동의 체크박스 추가. 약관은 [필수], 마케팅은 [선택]으로 표시 강화. 동의했으면 sessionStorage에 의도 저장 → `MarketingConsentApplier`(PendingInviteApplier 패턴)가 OAuth callback 후 `profiles.marketing_push_enabled = true` 반영.

### (3) 뉴스 목록 출처 항상 표시 — 저작권 안전망
`/news` 카드 하단에 `source_name` 노출. 이전엔 detail 진입해야 출처가 보였음. 자동 임포트 뉴스가 매일 들어오는데 목록에서 바로 출처 확인 가능해야 안전.

### (4) 만 14세 미만 보호자 동의 — 정보통신망법 시행령 §16
signup에 보호자 메일 신청 안내 (mailto deeplink with subject·body 프리필). terms 제5조에 14세 미만 가입 제한·법정대리인 신청 절차 신설. privacy **제5조의4 (만 14세 미만 아동의 개인정보 처리)** 신설 조항 — 가입 거절 절차·법정대리인 신청 절차·열람·정정·삭제 권리·사후 확인 시 자격 해지 4개 항목.

## 운영 메모

- **RSS는 빠른 발견 도구**. Feedly·NetNewsWire 자동 발견되니 외부 자동화에서 도시공존 콘텐츠 활용 가능. 다만 Google News Sitemap은 별도. 등재 신청 후 처음 4주는 트래픽 미미하더라도 정상.
- **동적 OG ↔ metadata.openGraph.images 충돌** 함정 주의. 명시적 images가 있으면 같은 폴더 `opengraph-image.tsx`가 무시됨. 동적 OG 쓰려면 명시적 images를 endpoint로 가리키게.
- **ConsentManager 패턴**: 한국 법규는 "쿠키 사용 자체"에 동의받기 요구라 Vercel Analytics가 익명이어도 동의 후 로드해야 안전. 한 컴포넌트에서 배너 UI + Analytics 게이팅 다 처리하면 깔끔.
- **MarketingConsentApplier ↔ PendingInviteApplier 패턴** 재활용. OAuth redirect 거치는 동의·코드는 sessionStorage에 두고 user mount 시 처리하는 게 가장 안정적.
- 14세 미만 안내 mailto에 subject·body 프리필해두니 보호자가 작성 부담 적음. URL encode 정성껏.
