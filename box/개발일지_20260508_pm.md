## 2. 갤럭시 플립 모달 잘림 이슈
테스터 제보 + 스크린샷:
> 갤럭시 플립에서 화면 잘림 이슈 있어, 전체적으로 잘 안 보임

### 원인
지도 페이지의 바텀 시트 모달들이 `max-h-[72vh]` 등 **`vh` 단위** 사용:
- 갤럭시 Z Flip은 22:9 세로 긴 비율 → vh 기준 모달이 꽉 차 보이지만
- 모바일 브라우저 chrome(주소창)이 동적으로 줄어들 때 `vh`는 갱신 안 됨 → 보이는 영역과 어긋남
- 결과: 모달 내부 콘텐츠가 잘려 보이거나 스크롤이 빡빡함

### 해결
지도 페이지 모든 바텀 시트를 `dvh`(dynamic viewport height) 단위로 전환 + 비율도 살짝 키움:

| 모달 | 기존 | 수정 |
|---|---|---|
| 고양이 카드 | `max-h-[72vh]` | `max-h-[85dvh]` |
| 동네 모달 | `60vh` / `calc(60vh - 80px)` | `70dvh` / `calc(70dvh - 80px)` |
| 채팅 모달 | `height: 55vh` | `height: 65dvh` |

### 교훈
- 모바일 풀스크린 요소(모달/지도/캔버스)는 처음부터 **`dvh` 사용**
- `vh`는 데스크탑 호환성 fallback이 아니라 그냥 모바일에서 부정확한 단위
- iOS 사파리·Android Chrome 모두 `dvh` 지원 (2023년 이후)

### 커밋
`fix: 갤럭시 플립 등 세로 긴 화면 모달 잘림 이슈 수정` (72c5b45)

## 3. 돌봄일지 등록 버튼이 안 보임
사용자 제보:
> 돌봄일지에서 엔터 버튼이 안 보이네

### 원인
`CareLogTab.tsx` 입력 폼:
- 메모 입력창 옆에 **w-9 h-9 작은 버튼 + Plus 아이콘** 1개로만 등록
- 사용자 멘탈 모델: 메모 쓰고 엔터 → 등록 / 또는 명확한 "등록" 버튼
- 작은 + 아이콘은 "추가" 의미로 안 읽힘 (특히 처음 쓰는 사용자)
- `onKeyDown` 핸들러도 없어서 엔터 눌러도 아무 반응 없음

### 해결
**시각 + 기능 동시 개선**:

1. 메모 옆의 작은 버튼 → **한 줄 전체 폭 "돌봄 기록 등록" 버튼**으로 분리
   - `Send` 아이콘 + 텍스트 "돌봄 기록 등록"
   - `bg-primary text-white text-[13px] font-extrabold`
   - 비활성 상태(careType 미선택)는 `disabled:opacity-40` 그대로

2. 메모 입력창에 **Enter 키 제출 핸들러** 추가
   ```tsx
   onKeyDown={(e) => {
     if (e.key === "Enter" && !e.nativeEvent.isComposing
         && careType && !submitting) {
       e.preventDefault();
       handleSubmit();
     }
   }}
   ```
   - **`!e.nativeEvent.isComposing` 필수**: 한글 IME 조합 중 엔터(글자 확정용)는 무시
   - 카카오 채팅과 동일한 방식

3. 좁은 화면에서 카메라 버튼 잘리지 않도록 `shrink-0` + 메모 input에 `min-w-0` 보강

### 교훈
- **아이콘 단독 버튼은 1차 액션에 부적합**: + 아이콘은 "옵션 펼치기" / "또 다른 항목 추가"로도 해석 가능 → 1차 등록은 라벨 텍스트 명시
- **한국 사용자는 엔터 제출을 강하게 기대**: 카카오톡·슬랙·검색창 모두 엔터 = 전송. 입력창에는 무조건 엔터 핸들러 + IME 조합 체크
- **w-9 h-9는 너무 작음**: 모바일 터치 타겟 최소 44x44 권장. 지금처럼 좁으면 등록 누르려다 카메라 누르기 쉬움

### 커밋
`fix: 돌봄일지 등록 버튼 잘 안보이는 문제 + 엔터키 제출 지원` (51ea7c2)

## 4. Play Console 비공개 테스트 운영 — 옵트인 링크
사용자 문의:
> 구글콘솔 비공개 테스트 링크 줬는데 "요청한 URL을 서버에서 찾을 수 없습니다" 떠

### 원인
공유한 링크가 일반 Play 스토어 URL(`play.google.com/store/apps/details?id=...`)였음. 비공개 테스트 앱은 아직 **공개 안 된 상태**라 일반 스토어 URL은 404.

### 정답: 옵트인 링크 형식
```
https://play.google.com/apps/testing/<package_name>
```
(`/store/apps/details`가 아니라 `/apps/testing/`)

### 비공개 테스트의 2단계 가입 흐름
1. 테스터가 **옵트인 URL** 클릭 → "테스터 되기" 페이지
2. **"테스터 되기"** 버튼 클릭 (이게 빠지면 안 됨)
3. 그 페이지의 "Google Play에서 다운로드" 링크로 설치

### 위치
Play Console → 테스트 → 비공개 테스트 → 테스터 탭 → "테스터가 가입하는 방법" → "웹에서 가입" → 링크 복사.

### 교훈
- 다음에 테스터 모집할 때 **카카오톡 공지 템플릿** 만들어 두기:
  > 1. 본인 Gmail이 등록되었는지 확인
  > 2. 옵트인 링크 클릭 → "테스터 되기"
  > 3. 그 후 "Google Play에서 다운로드" 링크로 설치
  > (일반 스토어 검색으로는 안 보입니다)
