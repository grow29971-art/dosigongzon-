### 초기 구현 (시뮬레이션)

- 1.2초 딜레이 + 타이핑 인디케이터 애니메이션

### 모달 안 열리는 문제 해결
- **원인**: 모달이 `<main max-w-lg>` 안에 렌더링되어 모바일에서 표시 안 됨
- **해결**: `createPortal`로 `document.body`에 직접 렌더링
- `z-[100]` (BottomNav z-50 위), `body overflow hidden` 스크롤 잠금
- 한글 입력 중 엔터 전송 방지 (`isComposing` 체크)

## 4. Gemini API 연동 — 실제 AI 집사

### 작업 내용
- `npm install @google/generative-ai` (v0.24.1)
- `app/api/chat/route.ts` — Next.js Route Handler로 Gemini 백엔드 API 생성
- `.env.local`에 `GOOGLE_GENERATIVE_AI_API_KEY` 설정
- `AIChatModal.tsx` — setTimeout 시뮬레이션 → `/api/chat` fetch 요청으로 변경

### 시스템 프롬프트
- 페르소나: 도시공존 AI 전문가 집사
- 친절한 말투, 성우님 호칭, 응원 멘트
- 500자 이내, 2~4문장 간결 답변

### 발생한 문제들 & 해결

#### 문제 1: 에러 원인 불명
- catch에서 실제 에러를 삼키고 고정 메시지만 표시
- **해결**: 디버그 모드 추가 — 서버에서 `debug` 필드로 실제 에러 전달, 프론트에서 표시

#### 문제 2: 429 Too Many Requests
- `gemini-2.0-flash` 무료 티어 쿼터 초과
- **해결**: 모델을 `gemini-1.5-flash`로 변경 + `maxOutputTokens: 300` 제한
- 429 전용 친절한 안내 메시지 추가

#### 문제 3: 404 Not Found — 모델 없음
- `gemini-1.5-flash`, `gemini-pro` 등 구 모델이 API에서 폐기됨
- 진단용 `app/api/models/route.ts` 생성하여 사용 가능 모델 34개 조회
- **해결**: 모델 자동 폴백 로직 구현
  - 1순위: `gemini-2.0-flash-lite` (가볍고 무료 티어 친화)
  - 2순위: `gemini-2.0-flash`
  - 3순위: `gemini-2.5-flash`

### 교훈
- `.env.local`은 서버 시작 시에만 로드됨 → 변경 후 반드시 서버 재시작
- Google AI 모델은 빠르게 폐기됨 → ListModels API로 확인 필요
- 에러는 삼키지 말고 프론트까지 전달해야 디버깅 가능
