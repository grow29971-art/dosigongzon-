## 1. 숏츠 무한 무작위 재생
사용자 피드백: "30개 다 보면 끝나서 식상함, 매번 다른 순서로 나왔으면."

해결 2단계로:
- **3사이클 확장**: `/shorts` 진입 시 영상 풀을 3번 독립 셔플해서 90개 카드 한 번에 마운트. 사이클마다 같은 영상이 다른 위치에 등장 → 무한 무작위 느낌.
- **카드별 동적 재셔플**: IntersectionObserver로 새 카드가 보이면 `onVisible(index)` 콜백 → ShortsFeed가 `items[index+1:]` 부분만 Fisher-Yates 재셔플. 이미 본 카드는 위치 고정(뒤로 스크롤해도 같은 영상). 다음 카드 예측 불가능.

키 충돌 방지: 같은 `short.id`가 여러 사이클에 등장하므로 React key를 `${i}-${s.id}` 복합 키로.

`/shorts/[id]` (deep-link 라우트)도 동일 처리 — target 첫 카드 + 핀고정 + 3사이클 셔플.

## 2. 전체 채팅 (지도 페이지)
기존 동네 채팅(현재 구 단위)에 더해 **모든 지역이 함께 쓰는 방** 추가.

- DB 변경 없음 — `area_chats` 테이블의 `area` 컬럼이 자유 텍스트라 `"전체"` 값으로 분리만 함.
- `chatArea` state 도입 — 기존 `currentGu`로 하드코딩된 부분 모두 `chatArea`로 일반화.
- 좌하단 FAB 두 개:
  - 🌐 **전체** (teal `#48A59E` 그라디언트, Globe 아이콘)
  - 💬 **{현재 구}** (기존 갈색)
- 채팅 패널 헤더가 스코프에 따라 "전체 채팅" 또는 "{구} 채팅" 표시.
- 폴링 effect의 `currentGu` 의존성도 `chatArea`로 교체.

`cleanup-area-chats` 주간 cron이 area_chats 전체를 비우니까 전체 채팅도 매주 월요일 새로고침됨 — 의도된 동작.

## 3. 시청 통계 메트릭 명확화
사용자: "전체 방문자 900명인데 어떻게 400명이 400편 봤다고 나옴? 이상함."

진단:
- "방문자 900명" = `daily_stats` 일별 IP-dedup 누적합 (같은 IP 다른 날 = 별도)
- "시청자 400명" = `shorts_view_events` localStorage UUID 고유 카운트 (디바이스 단위)
- 측정 단위가 달라서 직접 비교 불가. 400 < 900은 정상 (subset).
- "400명 = 400편"은 인당 평균 1편 — 첫 영상 보고 빠지는 사용자가 많은 자연스러운 결과.

코드 버그는 없음. 향후 표시 텍스트 다듬기 정도가 개선 후보 (지금은 "400명이 400편 시청"이 모호함).

또한 **외부 YouTube 시청은 우리 카운터에 반영 안 됨** — RPC 호출은 `dosigongzon.com/shorts` iframe 시청 시에만 발생. 주소창이 우리 도메인이고 iframe 안에 있을 때만 카운트.

## 4. Play Store 등록 준비 (대장정)

### 사이트 쪽 사전 작업
- `assetlinks.json` 패키지명을 `com.dosigongzon.app` → `kr.dosigongzon.app` 로 교체
  - 이유: Play Console에서 기존 패키지명이 "이미 사용 중" 거부됨. 누가 선점한 듯.
  - 대안 `kr.` 접두로 한국 도메인 기반 충돌 회피.
- `twa-manifest.json`, `app/build.gradle`도 같은 패키지명으로 일괄 수정
- 사이트 재배포 → live 검증 완료

### 자료 자동 생성 (Puppeteer)
이미 설치된 puppeteer로 자동화 파이프라인 3종 작성:
- `capture-screenshots.mjs`: 휴대폰 스크린샷 8장 (412×915 viewport, devicePixelRatio 2.625 → 1080×2400 고해상도) — `/shorts /` `/protection` `/protection/emergency-guide` `/tips` `/news` `/guide` `/about`
- `generate-feature-graphic.mjs`: 1024×500 피처 그래픽 — HTML/CSS로 그라디언트·발자국 패턴·"폰 목업" 디자인 + Pretendard 폰트, Puppeteer로 PNG 렌더
- `capture-tablet-screenshots.mjs`: 7인치(800×1280) + 10인치(1200×1920) 태블릿 스크린샷 각 6장씩

총 자료: 아이콘 1 + 피처 1 + 휴대폰 8 + 7인치 6 + 10인치 6 = **22장**.
바탕화면 `도시공존-플레이스토어-자료/` 폴더로 일괄 복사 → 사용자가 Play Console에 드래그.

## 가져갈 교훈
- **TWA 패키지명은 사이트의 `/.well-known/assetlinks.json`과 짝지어져 있음.** 패키지명 바꾸면 assetlinks.json도 동시에 갱신해야 검증 통과.
- **Play Store 일반 패키지명(`com.X.app`)은 의외로 충돌 잦음.** 도메인 기반 + 국가 코드 접두사(`kr.X.app`)가 안전.
- **Puppeteer로 스크린샷 자동화 ROI 큼.** 모바일·태블릿 다양한 사이즈 한 번에 생성, Play Store 자료 준비 시간 1/10로 단축.
- **TWA AAB 빌드를 로컬 환경 없이 PWABuilder.com으로 가능.** Java/Android SDK 설치 불필요.
- **Play Store 검수에서 "광고" 카테고리 체크 안 하는 게 중요.** 한 번 체크하면 광고 앱으로 분류돼 정책 훨씬 까다로워짐. 도시공존은 광고 0이라 정직하게 비워두면 됨.
- **숏츠 카드별 재셔플 패턴**: IntersectionObserver의 `onVisible` 콜백을 ShortsFeed 부모로 전파해서 부모가 items[index+1:]만 재셔플. 이미 본 카드는 마운트 유지(key 안정), 안 본 카드만 remount. 좋은 데이터 흐름 디자인.
