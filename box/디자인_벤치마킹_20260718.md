# 디자인 벤치마킹 — 잘 만든 앱 동선/배치 이식안 (2026-07-18)

> 요청: 이용자 많은 웹/앱의 유저 동선·배치를 조사해 도시공존에 이식.
> 방법: 디자인 에이전트 3명이 WebSearch/WebFetch로 실제 패턴 조사 + 우리 코드 대조.
> 정박점(실측): **활성화 = 가입 첫 세션이 전부(좀비 79%), 코어 루프 = 밥주기(care).** 커머스형 대형몰은 제외(곁길).
> ⚠ 비주얼 1:1 복제 아님 — 검증된 동선/위계 패턴만 이식. 브랜드·디자인 자산 도용 금지.

## 🎯 3개 리서치가 만난 단 하나의 결론
**"오늘 밥 줄 고양이 1탭 급식"을 앱의 중심축으로 — 온보딩 첫 행동 = 홈 최상단 = 반복 넛지, 전부 이걸로 통일.**
지금은 밥주기(코어)가 온보딩에도, 홈 위계에도 최상단이 아니다. 이게 좀비 79%·반복 미고착의 구조적 원인.

## 1. 온보딩 — "가입을 첫 행동 뒤로" (inverted onboarding)

| 앱 | 첫 액션 강제 방식 | 교훈 |
|---|---|---|
| Duolingo | 30초 내 1문제, 가입은 그 후. 먼저 "왜?"만 질문 | 가치 체험 → 가입 마찰 제거(리텐션 최대 임팩트) |
| Toss | 1화면 1질문(가입폼도 필드 1개씩) | 인지부하↓ 완주율↑, 단계별 이탈 계측 |
| 당근 | 진입 즉시 동네 설정이 관문 | 위치=개인화 스위치, 설정 순간 홈이 "내 동네"로 |
| Strava | 부가기능 다 스킵, "Record"만 | 첫 기록 = 진짜 activation(가입 아님) |
| 오늘의집 | 관심 태그 → 즉시 맞춤 피드 | 저마찰 선택 1번으로 "내 것 같은" 화면 |

*출처: userguiding/toss.tech/karrot help/uxcam 등 UX 분석·공식 도움말(2차 관찰, 내부지표 아님).*

### 우리 온보딩 재설계 초안 (감성 3슬라이드 → 행동 4스텝)
- **STEP1 감성 1장**: "우리 동네에도 밥을 기다리는 아이가 있어요" → `시작하기`(기존 3장→1장, 건너뛰기 유지)
- **STEP2 동네 1개 설정(당근式)**: LBS 신고 전이라 GPS 대신 **행정동 검색 칩**
- **STEP3 첫 행동 강제(Strava式, 가입 전)**: 그 동네 고양이 3장 → `🍚 밥 챙겨주기` 1탭 → 성공 피드백. 등록 0이면 "첫 등록자 되기"로 대체. 가입 전 pending care는 localStorage→가입 후 flush
- **STEP4 가치 체험 후 가입(Duolingo式)**: "방금 준 밥, 계정 만들면 기록으로 남아요" — 여기서만 가입 요구
- 재사용 자산: `app/onboarding/page.tsx`(SLIDES 축소), `FirstCheerCard.tsx`(catCount===0 훅), heartbeat 키프레임

## 2. 지도·밥주기 반복 루프 — 병목은 "동선 길이"가 아니라 "cue 부재"
※ 발견: 홈 1탭 밥주기(`MyCatsQuickCare.tsx`)는 **이미 구현돼 있음.** 그래서 넛지·streak·푸시에 집중.

| 앱 | 반복 유도 장치 |
|---|---|
| 카카오/구글맵 | 마커 1탭 → **바텀시트에 액션까지**(딥링크 0탭), 마커 44px+ |
| Strava | **주간 streak**(일간보다 지속률↑·죄책감↓), 소셜 검증 시 streak 5.69 vs 4.25일 |
| Duolingo | **2티어 푸시**(routine=습관창 gentle / save=끊기기 직전), 홈위젯 |
| 헬스앱 | 1탭 로깅 + 오늘 할당 진행바 |

*출처: trophy.so/smashing/logrocket/nngroup/duolingo blog 등.*

### 적용안(우선순위)
1. **지도 마커→바텀시트에 "밥 줬어요" 버튼**: 현재 5+탭 → 2탭. 시트 안에서 `createCareLog({care_type:'feed'})` 즉시. 상세는 "더보기".
2. **주간 streak + 오늘 할당 진행칩**: `MyCatsQuickCare`의 `doneCount`를 "오늘 3마리 중 1" + 주간 연속 배지로 승격(일간 아닌 주간 = 코어 리듬·죄책감↓).
3. **"밥 안 준 아이" 넛지(당근式)**: 홈/지도 상단에 오늘 미급식 담당묘("○○ 아직 안 줬어요"). GPS 없이 담당목록 기준.
4. **2티어 푸시**: routine(저녁 급식시간 gentle) / save(주간 streak 끊기기 직전). 기존 VAPID 재사용, **남발 금지 — 이 둘만.**
- 재사용 자산: `MyCatsQuickCare.tsx`(doneCount), `map/page.tsx`(마커 시트), `cats/[id]/page.tsx`, `CareLogTab.tsx`

## 3. 홈 정보위계 + 바텀네비 (현재 6탭: 홈/지도/AI집사/커뮤니티/쇼핑(공사중)/마이)

| 앱 | 홈 최상단 | 탭 원칙 |
|---|---|---|
| Toss | 개인화 요약 → 다음 행동 | 필요행동만 상단 |
| Duolingo | "오늘 할 일" 단일 경로 | 코어 액션 1개 최상단 고정 |
| 당근 | 지역 피드 | 5탭, 죽은 기능은 "전체 서비스" 서랍으로 |
| 오늘의집 | 피드+즉시행동 | 탭 늘리기보다 카드 안 완결 |

*원칙(NN/g·Material): 바텀탭 **3~5개 적정**, 6개는 터치타깃 좁아져 오탭. 죽은 탭은 숨김.*

### 재배치안
- **홈 최상단 = "오늘 밥 줄 담당 고양이" 1탭 급식**(기존 `hungryCatName` 로직 승격). 다마고치 히어로는 2번째로 강등(정서 훅이지 코어 액션 아님). 날씨/브리핑은 접기.
- **바텀네비 6→4탭: 홈·지도·커뮤니티·마이.**
  - 쇼핑(공사중) 제거 → 홈/마이 카드 "곧 오픈"으로 흡수(회의 결론).
  - AI집사 탭 제거 → 홈 상단 아이콘 진입점으로.
- 재사용 자산: `HomeAuthed.tsx`(히어로 순서), `BottomNav.tsx`(탭 배열)
