## 1. DB 스키마 최적화

### 정규화 판단
- `author_name/avatar/level/title` 비정규화 → **의도적 스냅샷 패턴, 변경 불필요**
- 이 규모에서 정규화(JOIN)보다 비정규화(중복 저장)가 읽기 성능에 유리

### 인덱스 추가 (11개)

|---|---|---|
| cats | region | 동 단위 그룹핑 |
| cats | caretaker_id + created_at | 마이페이지 내 고양이 |
| cats | hidden + created_at | RLS 필터 |
| cat_comments | kind (alert 부분 인덱스) | 경보 카운트 |
| cat_comments | author_id | 마이페이지 내 기록 |

### 인덱스 원칙
- **해야 하는 이유**: WHERE/ORDER BY 풀 테이블 스캔 방지
- **많이 하면 안 되는 이유**: INSERT/UPDATE 성능 저하 + 스토리지
- 현재 테이블당 2~3개 → 적정 수준

## 2. Next.js 서버 컴포넌트 적용

### 약품 가이드 페이지 전환
- `"use client"` 제거 → async 서버 컴포넌트
- `useEffect` + `useState` → 서버에서 직접 DB 조회
- `useRouter().back()` → 별도 클라이언트 컴포넌트(`BackButton.tsx`)로 분리
- `Link` 컴포넌트 (서버 호환)

### 효과
- JS 번들 감소 (BackButton만 클라이언트)
- 초기 로딩 즉시 (스피너 없이 SSR)
- SEO 개선 (내용 포함된 HTML)

### 서버 vs 클라이언트 구분
| 서버 컴포넌트 | 클라이언트 컴포넌트 |
|---|---|
| 데이터 읽기/표시 | 인터랙션 (useState, onClick) |
| DB 직접 접근 | 브라우저 API (지도, 채팅) |
| JS 안 보냄 | hooks 사용 가능 |

### 구성
- VAPID 키 생성 + Vercel env 추가
- `push_subscriptions` 테이블 (기기별 구독 저장)
- `app/api/push/subscribe/route.ts` — 구독 등록
- `app/api/push/send/route.ts` — web-push로 알림 전송
- `app/components/PushSubscriber.tsx` — 로그인 시 자동 구독
- `public/sw.js` — push/notificationclick 이벤트 핸들러

### 동작
1. 로그인 시 → 알림 권한 요청 → 구독 DB 저장
2. 쪽지 보내면 → `/api/push/send` 호출 → 상대방 기기에 알림
3. 알림 클릭 → 쪽지함으로 이동
4. 구독 만료(410) → 자동 삭제

## 9. 모바일 지도 로딩 수정

### 문제
- 모바일에서 지도가 "불러오는 중" 또는 "스크립트 로드 실패"

### 원인
1. 카카오 API 키가 `dosigongzon.com` 등록된 키와 불일치
2. 서비스 워커(PWA)가 이전 빌드를 캐싱 → 새 키 반영 안 됨
3. 스크립트 로드 감지 로직에서 이미 로드된 스크립트를 놓치는 버그

### 해결
- `NEXT_PUBLIC_KAKAO_MAP_KEY` 교체 (`89a876a...` → dosigongzon.com 등록 키)
- 서비스 워커 캐시 v1→v2 강제 갱신
- 스크립트 로드 감지에 폴링 체크 추가
