## §5. 크론 admin-daily-digest 결행 원인 확정
**증상**: admin-daily-digest(`10 0 * * *`) 발화 이력 7/14✅(00:37)·7/15✅(00:29)·**7/16❌**·7/17✅(00:44)·**7/18❌**. 발화해도 정시(00:10) 아닌 창 내 늦은발화, 40% 결행.

**원인 (코드 추적 + 웹확인)**:
1. **하트비트가 미들웨어(`proxy.ts` logCronRun)에 있음** → cron_runs 행 부재 = 라우트 실패 아니라 **Vercel이 URL을 아예 호출 안 함.** 앱 코드(디지스트 로직) 결백.
2. **Vercel Hobby 크론 = 하루 1회 한정 + 정시 보장 없음 + 배달 자체가 베스트에포트(누락 가능).** (vercel.com/docs 확인) 예약시각~다음 1시간 사이 랜덤 발화, 일부 누락.
3. **악화요인**: 00시대에 news-crawl(00:00)·digest(00:10)·payment-reconcile(00:20) 3개 클러스터 → 가운데 digest가 반복 희생.
4. **7/13 스케줄 이동(23:00→00:10)은 오진 처방** — 붐비는 00시대 창으로 밀어넣은 셈이라 결행 지속.

**해결 — 구현·검증 완료 (커밋 98ba42d, 배포됨)**:
- **`daily-dispatch`(0 0 * * *) 디스패처 신설** — news-crawl·admin-daily-digest·payment-reconcile를 병렬 호출(`Promise.allSettled`, 각 실패 격리). Hobby 보장 대상 3→1, 세 잡 동반 실행을 코드로 보장.
- vercel.json: 개별 3크론 제거, 디스패처 1개로(21→19). 기존 라우트는 그대로 두고 디스패처가 호출 → 서브 호출도 미들웨어 하트비트로 cron_runs 기록 유지(모니터링 보존).
- **검증(2026-07-18 01:30 UTC 수동 실행)**: 응답 3잡 전부 status 200·ok, 총 39초(60초 한도 내). cron_runs에 dispatch→admin-daily-digest·payment-reconcile·news-crawl 4행 즉시 기록 확인.
- 남은 옵션(미구현): payment-reconcile은 돈 관련이라 GitHub Actions 이중 호출까지 두면 더 안전. Pro 승급($20)은 WAU 6엔 과함.
- 롤백: daily-dispatch 제거 + 3개 개별 크론 복구.

**정정**: 어제 "engagement-push만 유령"이라 했으나, admin-daily-digest도 간헐 결행(플랫폼 배달 문제)으로 추가. payment-reconcile은 7/17·7/18 2일 연속 정상(회복).

- **daily-dispatch ✅ 배포·수동검증**(01:30 3잡 200 ok, 39초). **내일(7/19) 00:00 UTC 첫 자동발화**가 실전 검증 포인트 — 그때 news-crawl·admin-daily-digest·payment-reconcile가 dispatch 뒤 다 찍히면 결행 근본 해소 확인.

- **미확인(폴링 안 함, 다음 세션 확인)**: ①daily-dispatch 첫 자동발화(7/19 00:00 UTC) = 결행 근본해소 실전검증 ②일요일 배치(weekly-battle-payout·retention-report·weekly-digest·postcard, 7/19 23:00 UTC 창).
