## 1. 지도 마커 추가 최적화

사용자 제보: "마커 너무 많아서 렉." 어제 viewport 필터링 + 80개 상한은 적용했는데 여전히 무거움.

### 1-1. 줌 티어별 표시 강도 (가장 큰 win)
- **Tier 1** (level ≥ 9 광역 뷰): 작은 컬러 dot + 카운트 배지만, **사진 0장**
- **Tier 2** (level 6~8 중간): 사진 1장 + 동 + 카운트
- **Tier 3** (level < 6 동네): 풀 마커 (사진 3 + 동 + 카운트, 기존)
- 마커 상한도 차등 (200/120/80)

### 1-2. idle 200ms debounce
패닝 중 매 idle 마커 전체 재생성 → 멈춘 후 200ms 한 번만.

### 1-3. 마커 사진 Vercel Image Optimization 통과 (숨은 큰 win)
**기존**: `background-image: url(원본 1280px webp)` 직접 → 마커 48px인데 **1280px 다운로드**.
**변경**: `/_next/image?url=...&w=64&q=70` 통해 64~96px webp만.
**대역폭 ~10배 절감**.

### 1-4. Kakao SDK clusterer 라이브러리 추가
SDK URL에 `&libraries=services,clusterer` 추가. 향후 광역 뷰 자동 클러스터링 도입 시 추가 작업 0.

## 4. 안드로이드 앱 빌드 (Bubblewrap TWA) 🎉

사용자가 앱 가능성 물음 → "오늘 안드로이드만 가능" 답변 → 진행.

### 4-1. PWABuilder 시도 → 무한 로딩 실패
원인: `Cross-Origin-Resource-Policy: same-origin` 헤더가 PWABuilder의 외부 fetch 차단. 
**Fix**: `next.config.ts`에서 `/manifest.json`, `/.well-known/*`, `/icons/*`, `/sw.js` 만 `cross-origin` 허용 + `Access-Control-Allow-Origin: *`. 보안 헤더는 다른 경로엔 그대로 유지.

### 4-2. PWABuilder 재시도 → 여전히 무한 로딩
PWABuilder 측 SPA 분석 한계로 추정. **Bubblewrap CLI 로컬 빌드로 전환**.

### 4-3. PWA 보강
`manifest.json`:
- `lang: "ko-KR"`, `dir: "ltr"`
- `categories: ["lifestyle", "social", "utilities"]`
- `shortcuts: [지도, 커뮤니티, 보호지침]`
- `scope: "/"` 명시

### 4-4. assetlinks.json 준비
`public/.well-known/assetlinks.json` placeholder 생성. SHA-256 fingerprint는 빌드 후 채움.

### 4-5. Bubblewrap 셋업 (사용자 PowerShell)
환경 비어있던 상태에서:
- `npx @bubblewrap/cli doctor` — JDK 17 자동 다운로드(174MB) + Android SDK 자동 다운로드(76MB)
- `npx @bubblewrap/cli init --manifest=https://dosigongzon.com/manifest.json` — 인터랙티브 프롬프트
- 사용자가 Application name·Application ID·키스토어 정보 입력 (몇몇 필드에 비번을 위치/이름으로 잘못 넣었지만 동작엔 문제 없음)

### 4-6. 빌드 실패 → fix
**1차 에러**: `Could not reserve enough space for 1572864KB object heap` (JVM 메모리)
**해결 시도 1**: `gradle.properties`에 `-Xmx1024m` 추가 → 안 먹힘 (Bubblewrap 자체 인자 주입)
**해결 시도 2**: 환경변수 `GRADLE_OPTS` → 안 먹힘
**해결 시도 3**: `Set-Content` PowerShell here-string → 사용자 입력 꼬임
**해결 시도 4**: 메모장 직접 편집 → **성공**. `-Xmx512m` + `org.gradle.daemon=false`로 빌드 통과

### 4-7. 빌드 성공
- `app-release-signed.apk` (테스트 설치용)
- `app-release-bundle.aab` (Play Store 업로드용)

### 4-8. SHA-256 추출
`bubblewrap fingerprint list` → 빈 결과
**우회**: keytool 직접 호출 → `BE:C4:A5:02:F2:4D:38:34:F2:4C:C9:53:C7:D5:06:DA:C2:1C:E7:B2:A5:A4:AF:A1:D7:BD:D3:B0:52:23:FC:D3` 추출

### 4-9. assetlinks.json 실제 fingerprint 적용 + 배포
`public/.well-known/assetlinks.json` 갱신 후 `vercel --prod`. 검증으로 실제 값 확인 완료. **앱과 도메인 신뢰 연결 완료** — 앱에서 URL 바 안 뜸.

### 4-10. 남은 단계 (내일)
- Google Play Console $25 결제 + 가입
- AAB 업로드
- 스토어 자료 (피처 그래픽 1024×500 + 스크린샷 2장 + 설명문) 준비
- 검토 제출 → 1~3일 후 게시
