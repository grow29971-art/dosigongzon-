## 3. 돌봄 일일 cue 푸시 크론 — care-cue (커밋 `109147b`)

진단: 1탭 카드 깔아도 **앱을 열게 하는 트리거가 없으면 무용지물**. 기존 streak-reminder는 streak≥2 대상만 → 신규/끊긴 다수 누락.

**적용**: `/api/cron/care-cue` 매일 18:00 KST(`0 9 * * *`). 필터 = 마케팅 푸시 동의 + 고양이 보유 + 최근 7일 돌봄 기록 0건(streak-reminder와 audience 분리). 메시지 → `/` (홈 MyCatsQuickCare 카드).

**사전 검증 (실제 발송 X)**:
```
마케팅 푸시 동의:        29명
└ 고양이 보유:           8명
  └ 최근 7일 돌봄:       0명 (제외)
    → 최종 대상:         8명
       └ 푸시 구독자:    2명 (실 도달)
```

**파일**: `app/api/cron/care-cue/route.ts`, `vercel.json`.

## 5. 진단 — 왜 가입은 하는데 사용은 안 할까

데이터를 정직하게 짚어 가장 큰 원인 3개:

**① 광고 audience vs 앱 가치의 미스매치** (가장 큼)
- 앱은 *이미 길고양이를 돌보는 사람*용 도구. 길고양이 정기 캐어테이커는 인구 비율 매우 niche.
- Meta 광고는 broad audience("창립멤버 타이틀" 호기심)를 데려옴. 들어와서 "내가 뭘 해야?" — *"실제로 길고양이 챙기세요"*가 정답인데 광고만으론 안 됨.
- 주 38명 가입 → 8마리 등록(21%). 나머지 79%는 진짜 돌봄 인구가 아닌 거.

**② 콜드스타트 (저밀도) — 내 동네가 죽어보임**
- 243명이 전국 분포 → 신규의 본인 동네는 거의 비어있음. *"우리 동네 고양이 0~1마리"*가 죽은 인상을 줌.
- FirstCheerCard·도감·동네소식도 *동네에 콘텐츠가 있어야* 작동.

**③ 첫 1분 안에 "오 재밌네" 순간 부재**
- 인스타·틱톡: 가입 즉시 무한 콘텐츠 → 도파민. 우리: 가입 즉시 *네가 첫 행동을 일으켜야* 콘텐츠 생성. 일감 같은 인상 → 61% 원샷 이탈.

**더 깊은 한 줄**: *가입이 너무 싸고 핵심 행동(돌봄)이 너무 비싸다*. 카카오/구글 2탭 가입 vs 실제 길고양이 챙기는 행동 = 누수 구조.

## 6. 서클 사용자 0명 — 출시 후로 미루기

질문 *"서클 기능 사용자 몇 명?"* 데이터 직접 조회:

```
전체 가입자: 243명
─────────────────────
circles 테이블:        0행      ← 아무도 서클 안 만듦
circle_members:        1행 (pending) — 미수락
─────────────────────
서클 만든 사람:        0명
멤버를 1명+ 가진 owner: 0명
초대 수락한 사람:      0명
```

**판단**: 서클은 *학대 우려 아이 비공개* 후순위 기능. 출시 임박 + 더 큰 누수(돌봄 0·옵트인 12%)가 있어 **출시 후로 미루기**. 활성화하려면 첫 등록 시 *공개 범위 단계에서 "내 서클" 선택 → 그 자리에서 멤버 초대*로 흐름 자연스럽게 만드는 게 정석이지만 D-2엔 위험.

## 7. 의도 1문항 — welcome 끝 audience 흡수 (커밋 `497ae8d`)

진단의 ① 미스매치를 *코드로 흡수*. welcome 4 슬라이드 후(다음 파라미터가 기본 `/`일 때만) 3-way 의도 picker:

- 🐾 **이미 돌보고 있어요** → `/map` (지도·돌봄일지로 바로)
- 💛 **관심 있어 들어왔어요** → `/` (홈에서 천천히)
- 🎬 **그냥 구경하러** → `/shorts` (콘텐츠부터)

선택은 `auth.updateUser({data:{intent}})`로 user_metadata에 저장 → 추후 분포 분석. `next` 명시적이면 그대로 존중(URL 의도 보존). 건너뛰기 유지.

**파일**: `app/welcome/page.tsx`.

**측정**: 1~2주 후 진단 스크립트에 intent 분포 추가 → 광고가 어떤 audience를 데려오는지 정량 파악(예: caretaker 5% / interested 60% / browsing 35%면 광고 타게팅 좁힐 명확한 근거).

## 솔직한 평가

- **출시 직전 더 쌓는 건 위험 대비 가치 떨어짐.** 측정 대기가 합리적이고 그 사이 직접 점검(로그인해서 새 카드들 동작 확인)이 더 큰 가치.
- **audience 미스매치는 코드만으론 100% 못 해결.** 광고 작업이 병행돼야 함.
- 푸시 옵트인 12% (또는 cat-owner 25%)가 진짜 병목. PushCareCueOptIn이 깔렸지만 효과는 측정 후 판단.
- 한 세션에 너무 많이 깐 것도 사실(홈에 카드가 빽빽). 효과 없는 건 회수할 각오 필요.
