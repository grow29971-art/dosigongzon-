## ✅ 만장일치 합의

1. **동결 유지. 단 "대기"가 새 회피처가 되면 안 됨** (악마) — 동결 기간 = 인터뷰 준비 + 부채 청산 기간.
2. **판정 기준 선등록**: 답장을 열어보기 전에 "무엇이 나오면 A를 만들고, 무엇이 나오면 접는다"를 문서로 박제. 사후 해석 조작 차단.
3. **72시간 룰**: 3일 내 답장 3통 미만이면 쪽지 재발송이 아니라 코어 7명 한정 직접 접촉(전화/통화 심층 인터뷰)으로 전환.
4. **인터뷰는 가설 생성용, 판정은 행동 데이터로** (리서치·디자이너·BM 일치):
   - 유저의 기능 요청 = 니즈가 아니라 유저가 추측한 해결책. "그게 있으면 언제·뭘 하려고? 지금은 어떻게? 뭐가 제일 불편?"으로 번역.
   - 잠수자 답의 핵심은 "뭐가 없어서"가 아니라 **"대신 무엇을 쓰는가"**. 대체재가 '없음'이면 수요 자체가 약한 것.
   - 빌드 기준: 잠수 응답자 3명+ 이 유도 없이 같은 문제를 자발 언급 **AND** 코어 7명의 행동 데이터와 일치할 때만.
   - 활동 8명의 답에서 공통 동사 1개가 나오면 그게 코어 JTBD. 답이 제각각이면 제품이 아니라 운영자 개인에 붙은 것 → 커뮤니티 리더십 모델 검토.
   - 회수율 30% 미만이면 침묵 자체가 데이터(관계 자산 약함).

## 📌 관점별 핵심 발언 요약

- **BM**: 진짜 자산은 317명이 아니라 코어 7~8명+실주인 55명. 쪽지로 끝내지 말고 활동 유저 3~5명은 30분 통화로 승격. 돌봄자 개인은 지불력이 약해 B2C 과금은 구조적으로 어려움 — 지불자(단체·병원·지자체·브랜드) 탐색 병행.
- **리서치**: 코호트 W1%보다 "코어 7명은 첫 세션에 뭘 했고 이탈자는 뭘 안 했나"(활성화 이벤트 상관)가 먼저. 추가 쿼리 2개 — (a) 좀비 퍼널(317명 중 활동 0건 비율+가입→첫활동 시간) (b) 코어 7명 행동 해부(기능 구성비·요일·시간대).
- **악마의 변호인**: 앱내 쪽지 회수율 현실치는 10~20%(2~4통). 코어 7명이 지인·품앗이인지 검증 안 됨 — n=7은 자산이 아니라 표본오차일 수 있음. 자기기만 1순위 = "인터뷰 대기"를 면죄부로 쓰는 것.
- **한국심리**: 리마인드는 1회 엄수(2회부터 차단), 발송 4~5일 후 평일 저녁 8~9시. 재촉이 아니라 "○○ 요즘 잘 지내나 궁금해서요 :) …한 줄이라도 남겨주시면 다음 업데이트에 꼭 반영할게요" — '내 의견이 반영된다'는 주인공 의식. 예의상 답변("좋아요~")은 무신호, 구체 명사·시간·불만이 진짜 신호. 후속 질문은 "마지막으로 열었을 때 뭐 하려고 여셨어요?"(행동 회상). **코드 없는 최강 리텐션 = 주 1회 운영자가 코어 8명의 활동을 직접 목격·언급하는 쪽지**("어제 ○○ 밥 주신 거 봤어요") — 8명이면 10분.
- **도메인**: 이탈 가설 3개 — ① "등록=완료" 심리(등록은 1회성 목적 달성, 돌봄은 오프라인) ② 긴급성 미스매치(아플 때·민원·TNR 등 사건 발생 시에만 필요) ③ 동네 밀도 부족(반경 1~2km에 이웃 없으면 동네 카톡방으로 회귀). 대체재 없는 가치 = 지도 기반 개체별 케어 타임라인 + 비난 없는 소규모 안전지대. 무코드 행동 = 구청 동물복지팀+지역 TNR 조직 1곳 대면 청취.
- **B2G**: 지금의 매출 대안으로는 환상, 12~18개월 트랙으로는 유효(7~10월 제안 → 내년 본예산 → 2027 집행). 동결 중엔 주 2~3시간 자산 축적만: 시드 분리 실데이터셋(○○구 N마리), 월간 활동 리포트 1장, 적법 처리 증빙. **행정 우선순위: LBS 신고 > 통신판매업**(LBS 미신고는 형사 리스크+B2G 즉시 결격).
- **디자이너**: 잠수 가설 — "등록 후 할 일 없음"(다음 행동 미설계), "빈 방"(내 고양이·글에 반응 0), "현장과 단절"(기록 비용 > 가치). 준비 작업 = 코어 8명 행동 시퀀스 실측(읽기전용 SQL)으로 진짜 코어 루프 특정.
- **개발**: P0 = 미실행 SQL 소진(위 프로브로 확정 후). P1 = box/ SQL 193개 대장화(적용/미적용/폐기), cron_runs·토스 웹훅 검증. 빠른 빌드 대비 = 실측 SQL을 크론으로 자동화(daily_stats 적재) — 계측이므로 동결 위반 아님. tsc 통과, 타입 부채 없음.
