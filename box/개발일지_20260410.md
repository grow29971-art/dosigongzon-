## 3. 학습 로드맵 문서화

### 배경
- 외부 강의 커리큘럼 PDF("팔리는 제품을 위한 8주 파운데이션 — Next.js + Supabase + tRPC") 스크린샷 4장을 PRD/TRD에 추가 요청.
- 그대로 붙이면 PRD(제품 문서)와 성격 충돌 + 커리큘럼의 tRPC가 TRD(Drizzle + Route Handler)와 기술적 충돌.

### 결정
- **학습 로드맵은 새 파일로 분리**하고, TRD에는 shadcn/ui만 공식 편입.
- tRPC는 학습 목적으로만 진행, 본 프로젝트 도입은 v2 시점에 재평가.

### 신규 파일
- **`box/학습로드맵_8주파운데이션.md`**
  - 8주 커리큘럼 전체 (1~8주 주제/핵심 수업/과제)
  - 1개월차: TypeScript + shadcn/ui + 컴포넌트 조립 + RHF/Zod
  - 2개월차: Supabase 스키마 + tRPC + useQuery/useMutation + Vercel 배포
  - 도시공존 프로젝트와의 연관성 섹션 (이미 도입된 항목 vs 학습 후 검토할 항목)
  - 7주차 과제 일부 및 8주차 내용은 이미지에서 잘려 있어 "이미지에서 일부 잘림"으로 명시

### TRD v1.2 부분 갱신
- 헤더에 연동 문서로 `학습로드맵_8주파운데이션.md` 추가
- "부분 갱신(2026-04-10)" 주석 추가 (shadcn/ui 편입 사유 + tRPC 미도입 사유)
- **1.1 MVP 스택 표에 `UI Kit | shadcn/ui` 한 줄 추가** — 이유 칸에 세이지/클레이 팔레트 적용 자유로움, 블록 단위 점진 도입 근거 명시

### PRD는 건드리지 않음
- PRD는 제품 요구 문서라 학습 커리큘럼이 들어갈 자리가 아니라고 판단.
