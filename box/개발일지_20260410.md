3. **middleware가 뭔가 하나?**
   - Next.js 16 — `middleware.ts`가 아니라 `proxy.ts` (Next.js 16 리네임)
   - `proxy.ts`는 Supabase 세션 쿠키만 갱신, 라우팅에 영향 없음

### 실제 원인 — DevTools Elements 탭에서 확정
- 사용자가 보낸 스크린샷 URL이 `omega.vercel.app/?code=1eac04fa-...` 였음.
- **localhost:3000 이 아니라 프로덕션 URL이었음.**
- 흐름:
  1. 로그인 전: 사용자는 `http://localhost:3000` 에서 **새 코드**(6탭 with 지도)를 봄
  2. 로그인 버튼 클릭 → OAuth → Supabase → 공급자 redirect
  3. Supabase에 등록된 Redirect URL은 프로덕션 URL(`omega.vercel.app` = `city-amber-omega.vercel.app`)만 있었음
  4. OAuth 성공 후 사용자가 프로덕션으로 튕김
  5. 프로덕션에는 아직 **옛 코드**가 배포돼 있어서 5탭, 지도 없음
- **"로그인 후 지도가 사라진다"는 실제로는 "도메인이 바뀐다"** 가 정답.

- 이제 OAuth 로그인해도 localhost로 돌아옴 → 6탭 유지됨 확인.

### 배움

### 신규 SQL: `box/supabase_cat_comments_schema.sql`
```sql
create table public.cat_comments (
  id uuid primary key default gen_random_uuid(),
  cat_id uuid not null references public.cats(id) on delete cascade,
  author_id uuid references auth.users(id) on delete set null,
  author_name text,
  body text not null,
  kind text not null default 'note' check (kind in ('note', 'alert')),
  created_at timestamptz default now() not null
);
```

## 저녁 세션 배움
- **외부 서비스 설정 이슈는 연쇄로 터진다.** 하나의 기능(`/map`)을 움직이게 하는 데 카카오 콘솔 3곳(JS 키 / SDK 도메인 / 제품 활성화), Supabase 3곳(cats 테이블 / cat_comments 테이블 / Storage 버킷 / Redirect URL), Vercel 2곳(env var / project link) 의 설정이 필요했다. 코드는 이미 준비돼 있었는데도 이 "설정 체인"이 배포까지 이어지는 시간이 훨씬 길었다. 체크리스트 문서화 필요.
- **"브라우저가 이상해"를 의심하기 전에 주소창 URL부터 확인.** 캐시 디버깅에 15분 쓴 뒤에야 사용자가 보내준 스크린샷 URL에서 `omega.vercel.app` 이 보였다. OAuth 리다이렉트가 배포된 프로덕션 URL로 튕긴 거였다. 원인 진단의 가장 낮은 층(네트워크·URL)을 먼저 확인하는 습관이 필요.
- **Next.js 16에서 middleware의 이름은 `proxy.ts` 다.** `middleware.ts` 가 아님. 본 프로젝트도 `proxy.ts` + `lib/supabase/proxy.ts` 구조로 되어 있었고 이는 정상. (CVE-2025-29927 대응 차원의 리네임)
- **`next/script` 의 `onLoad`/`onError` 는 Next.js 16 + Turbopack 에서 믿을 게 못 된다.** `useEffect` 로 직접 `<script>` 주입하는 게 안정적. 이번 건에서는 결과적으로 카카오 제품 미활성화가 근본 원인이었지만, 디버깅 속도 면에서 직접 로드가 훨씬 명확했다.
- **더미 데이터는 CTE + UNION ALL 패턴**이 깔끔하다. 샘플 행의 UUID를 이름으로 조회해서 연관 테이블에 삽입하는 패턴을 이번에 처음 써봤는데, 여러 부모 행에 대해 각각 다른 자식 행을 꽂을 때 `DO $$ ... $$` 블록보다 가독성이 좋았다.
