## 2. otp_expired 에러 친절 안내 (`ff26212`)

admin/auth-errors에 `otp_expired` 3시간 전 8회 발생. 비밀번호 재설정 메일 1시간 만료·단일 사용 정책 때문.

**문제:** Supabase가 `otp_expired`를 `access_denied` 쿼리로 감싸서 보내는데, 기존 로직은 `access_denied`부터 매칭해 "로그인이 취소됐어요"라는 엉뚱한 메시지 표시.

**fix:** `lib/auth-errors.ts`에서 `otp_expired` / "email link is invalid" / "has expired" 조건을 `access_denied`보다 먼저 체크.

→ "재설정 링크가 만료됐어요 / 1시간·1회 제한 / 메일 다시 받기" 안내로 변경.

Pretendard CDN DNS·TLS 핸드셰이크 미리 끝냄 → 텍스트 깜빡임 감소.

### 4-2. supabase_index_optimization_v2.sql — 누락 인덱스 8종
쿼리 패턴 재분석 결과 1차 SQL에 빠진 핫 인덱스들:

- `cat_comments(cat_id, created_at desc)` — 고양이 상세 댓글 (18회 쿼리)
- `care_logs(cat_id, logged_at desc)` + `care_logs(author_id, logged_at desc)`
- `user_follows(follower_id)` + `user_follows(following_id)`
- `cat_likes(cat_id)` + `cat_likes(user_id)`
- `user_activity_regions(user_id)`
- `auth_error_logs(created_at desc)`

**N+1 쿼리 스캔 결과:** `lib/*-repo.ts` 10개 파일 전부 배치 조회 + Map lookup 패턴. N+1 없음 ✅.

### 4-3. SQL 실행 중 에러 → 수정

## 5. 통합 검색 기능 (`fdc0085`)

### 5-1. `/api/search` — anon 클라이언트 + RLS 필터
```
GET /api/search?q=검색어&type=all|cats|posts|hospitals|guides
```

- **cats**: name·description·region (hidden 제외)
- **posts**: title·content (hidden 제외)
- **rescue_hospitals**: name·address·district (hidden 제외)
- **guides**: 하드코딩 메타데이터 기반 키워드 매칭 (9종 가이드)

보안·성능:
- IP당 분당 30회 rate limit
- 2자 미만 거절 (tooShort 플래그 반환)
- 64자 초과 자르기
- ilike 패턴 `% _` 이스케이프 (LIKE injection 방어)

### 5-2. `/search` 페이지 UI
- **Debounce 350ms** — 입력 멈추면 요청
- 탭 5개 (전체·고양이·게시글·병원·가이드) + 결과 수 표시
- 4개 상태 분기: loading / empty / too-short / no-result
- URL 동기화 (`?q=xxx`)

### 5-3. 진입점 + 색인 정책
- 홈(HomeAuthed) 헤더 우측 아이콘 그룹에 돋보기 추가
- 커뮤니티 헤더 우측에도 돋보기 추가
- `robots.ts`에 `/search` disallow (파라미터 동적 결과라 색인 가치 낮음)

KST 주간 시작일 계산:

const day = nowKst.getDay(); // 0=일
const daysFromMonday = (day + 6) % 7;
nowKst.setDate(nowKst.getDate() - daysFromMonday);
