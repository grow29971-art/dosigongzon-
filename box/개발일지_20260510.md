## 2. 1차 최적화 — 채팅 1초 폴링 → Realtime + 30초 폴백 (커밋 `2cb7009`)

### 진범 발견
`app/(main)/map/page.tsx`에 다음 코드:
```ts
const interval = setInterval(fetchMessages, 1000);
```
- `area_chats` 테이블을 1초마다 `select("*").limit(50)` 폴링
- 50메시지 × 250바이트 ≈ 12KB/초 → **1유저가 1시간 켜두면 ~50MB**
- 5명 × 7일 누적 시 **수십 GB 발생 가능**

### 수정 패턴
다른 페이지(messages, community/[id])에서 이미 쓰는 **Realtime + 폴백** 패턴으로 통일:
1. `supabase.channel(...).on("postgres_changes", {filter}, cb).subscribe()` — websocket 단일 연결, 메시지당 ~0KB
2. 30초 폴백 polling (Realtime 끊김 대비)
3. `document.visibilityState === "visible"` 체크 → 백그라운드 탭에서는 폴링 중지
4. `visibilitychange` 이벤트로 탭 재활성 시 즉시 동기화

### 교훈 (메모리에 저장)
- **postgres_changes 가능한 테이블에는 짧은 인터벌 폴링 절대 금지**
- 메모리: `feedback_supabase_realtime_polling.md`

## 3. 1차 최적화 — Supabase Storage 이미지 변환 (커밋 `f232a89`)

### 진짜 큰 거 발견
지도 마커가 cat.photo_url 원본을 그대로 background-image로 사용:
- 200마리 × 4MB(원본) = **한 번 지도 로드에 800MB potential egress**
- `lib/cats-repo.ts`에 `thumbnailUrl()` 헬퍼가 이미 있었는데 안 쓰고 있었음 ()
- 코드 주석에 "Vercel Image Optimization은 background-image에선 400 발생"이라고 적혀 있어서 그냥 원본을 사용했던 것 — 실제로는 Pro의 Supabase 변환 endpoint는 background-image에서 잘 작동

### 변환 적용
모든 cat.photo_url, avatar_url을 Supabase 변환 endpoint(`/storage/v1/render/image/public/`)로 라우팅:

| 위치 | 변환 폭 | 효과 |
|---|---|---|
| 지도 마커 | width=128~192 (retina) | 4MB → ~10KB |
| 동네 모달 썸네일 | 96px | 4MB → ~10KB |
| 선택 고양이 상세 | width=800 | 4MB → ~50KB |
| 라이트박스 | width=1200, q=80 | 4MB → ~150KB |
| 댓글/채팅 아바타 | 40~56px | 1MB → ~5KB |
| 돌봄일지 사진 | width=300 | 4MB → ~30KB |

`optimizedImageUrl(url, width, q)` 헬퍼 신규 추가 (`lib/cats-repo.ts`).
모든 신규 처리 위치에 `loading="lazy"` + `decoding="async"`.

## 7. assetlinks.json Play App Signing 키 추가

### 배경
TWA(Trusted Web Activity) 앱 실행 시 상단 주소창 표시 → 진짜 앱처럼 안 보임 → 완성도 ↓.
원인: `/.well-known/assetlinks.json`의 SHA-256이 **Play 재서명 키**와 일치해야 하는데, 기존엔 **업로드 키**만 들어있었음.

### 처리
Play Console → 앱 무결성 → 앱 서명 → SHA-256 가져와서 추가:
```
A1:C5:5E:50:5F:26:DA:DF:FF:63:F8:8A:D1:7D:C9:23:46:D8:A7:0B:91:65:E9:C9:FF:04:B8:8B:AF:AF:67:F6
```

### 검증
Google Digital Asset Links API로 직접 검증:
```
GET https://digitalassetlinks.googleapis.com/v1/statements:list
    ?source.web.site=https://dosigongzon.com
    &relation=delegate_permission/common.handle_all_urls
```
응답: 두 fingerprint 모두 statements에 포함, `maxAge: 3599s`(1시간 캐시).

### 반영 시점
- Android는 1시간 캐시 → 새로 설치하는 사람은 즉시 적용
- 이미 설치한 테스터는 앱 데이터 삭제 또는 1~2시간 대기 필요 (로그인 풀림 주의)

## 8. 차단 유저 댓글 숨김 (커밋 `8c482ea`)

`/map` 고양이 댓글, `/community/[id]` 게시글 댓글 둘 다 동일 패턴:
```ts
const [list, blocked] = await Promise.all([
  listComments(id),
  getMyBlockedIdSet(),
]);
const filtered = blocked.size === 0
  ? list
  : list.filter((c) => !c.author_id || !blocked.has(c.author_id));
```
- 차단 등록 → 1분 캐시 TTL → 다음 댓글 fetch 시 자동 반영

- **Free 플랜 임계 모니터링 필수** — Supabase Reports에 일별 egress 보드 자주 확인. 1GB/일 넘으면 빨간불.
