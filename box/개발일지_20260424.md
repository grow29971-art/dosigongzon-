## 1. 고양이 위치 수정 시 함수 시그니처 에러 (`a751f33`)

이용자 제보 (카톡 + 스크린샷):
> "수정 실패: function public.calc_distance_m(double precision, double precision,
> double precision, double precision) does not exist"

**원인:**
- `cats.lat/lng` 컬럼이 `double precision`인데 `calc_distance_m` 함수는 `numeric` 시그니처로 정의
- Postgres는 함수 인자 타입 매칭이 엄격해서 암묵 캐스트 X
- `cats` UPDATE 시 트리거 `log_cat_location_change()`가 함수 호출 → 매칭 실패

**fix:** `box/supabase_calc_distance_m_fix.sql`
- `double precision` 버전 함수 overload 추가 (기존 `numeric` 버전 유지)
- Supabase SQL Editor에서 사용자가 수동 실행
- 실행 후 위치 수정 정상 작동 확인

## 2. /areas/[gu]/[한글동] 24개 페이지 500 에러 (`a751f33`, `30bdc90`)

Google Search Console에서 발견:
- 4/19~4/21 크롤링 24개 동별 페이지 모두 5xx
- 색인 실패 + 사이트맵에 등록된 URL이라 SEO 영향 큼

### 2-1. 1차 시도: Promise.allSettled + 안전한 fallback
- 한 쿼리 실패해도 빈 데이터로 폴백
- 개별 실패는 `console.error`로 Vercel 로그에 기록
- → 결과: 여전히 500

### 2-2. 진짜 원인 발견 (Vercel 로그)
```
TypeError: Invalid character in header content ["x-next-cache-tags"]
  code: 'ERR_INVALID_CHAR'
```

- Next.js ISR이 `x-next-cache-tags` 헤더에 페이지 경로 정보를 set
- 한글 동명("역삼동", "서빙고동" 등)이 비ASCII 문자라 HTTP 헤더에 들어가면 안 됨
- /areas/[slug]는 영문 slug라 멀쩡, [dong]만 영향
- 4/23에 빈 `generateStaticParams()`로 ISR on-demand 생성으로 바꾼 게 트리거

### 2-3. fix
```ts
export const dynamic = "force-dynamic";  // ISR 끄고 매 요청 SSR
```
- cache tag 자체를 안 만들어 헤더 문제 회피
- 매 요청 Supabase 쿼리 3건 추가지만 트래픽 적은 동별 페이지라 부담 작음
- 데이터 항상 fresh, sitemap·SEO 그대로

### 2-4. 검증
4개 대표 URL 전부 200 OK 회복. Search Console에서 "수정사항 검증" 시작 → 며칠 내 자동 재크롤링.

## 3. 위치 옮겼는데 동 이름이 이상한 문제 (`b7581db`)

이용자 제보 (이현창님):
> "장안동 28-7번지로 위치를 옮겼는데 송죽동으로 떠가지고ㅠ"

**원인:**
- Kakao `coord2RegionCode`는 같은 좌표에 대해 두 가지 동 정보 반환:
  - `result[0]`: 법정동(B) — 등기·주소 기준
  - `result[1]`: 행정동(H) — 주민등록·실생활 기준
- 기존 코드는 `result[0]`만 사용 → **법정동**이 들어감
- 수원시 장안구 장안동 28-7의 경우:
  - 법정동: "송죽동"
  - 행정동: "장안동"
- 사용자는 보통 행정동 이름으로 동네 인식 → 혼란

**fix:** 3개 컴포넌트 일괄 수정
- `CatLocationPicker.tsx` (위치 변경 모달)
- `AddCatModal.tsx` (고양이 등록)
- `mypage/activity-regions/page.tsx` (활동 지역 등록)

```ts
const admin = result.find((r) => r?.region_type === "H");
const legal = result.find((r) => r?.region_type === "B");
const target = admin ?? legal ?? result[0];
```

`map/page.tsx`의 `detectGu`는 region_2depth_name(구) 가져오는 거라 H/B 동일 → 수정 불필요.

## 5. 전국 광역시·도 SEO 확장 (`8e72b62`)

이용자 제보가 수원 거주자였던 것에서 출발. 현재 SEO 랜딩이 서울 25구만이라
다른 도시 사용자가 검색에서 도시공존을 발견할 통로 없음.

### 5-1. `lib/korea-regions.ts` 신규

DB의 `cats.region` 컬럼이 자유 입력이라 부분 매칭(ilike) 필요해서.

### 5-2. `/regions` 인덱스 페이지

- ItemList JSON-LD

### 5-3. `/regions/[sido]` 시·도별 랜딩

- `getSidoData`: matchKeywords로 OR 매칭한 cats + 인근 병원
- Promise.allSettled로 안전한 fallback (4/23 dong 페이지 교훈)

### 5-4. sitemap·진입점
