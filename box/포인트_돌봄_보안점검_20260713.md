# 포인트·돌봄 시스템 보안 점검 (2026-07-13)

포인트(현금성 할인) 도입으로 돌봄·출석 체계가 공격 표면이 됨에 따라 전수 점검.

## 가치 사슬과 상한

```
일일 출석체크(1회/일, 서버 강제) → checkin_days → 주간 마일스톤 → 최대 600P/주·계정
포인트 사용: 결제 승인 시 서버 재계산·원자 차감 → 1P = 1원
```
**구조적 상한**: 어떤 방법으로도 계정당 주 600P(600원) 초과 적립 불가.

## 잔여 리스크 (수용 또는 모니터링)

1. **다계정 파밍**: 계정당 600원/주 상한 + Turnstile 가입 방어 + 7일 개근 노동 필요
   → ROI가 낮아 저위험. 모니터링: point_ledger에서 주간 지급 상위 계정 정기 확인.
   악화 시 옵션: 가입 7일 후부터 포인트 사용 가능 / 휴대폰 인증.
2. **돌봄 기록의 질**: 한도 내(하루 150건) 무의미 기록으로 랭킹 상위 노리기 가능
   → 랭킹은 명예 보상이라 저위험. 악화 시 랭킹 산정을 "고양이 수 × 일수" 가중으로.
3. **출석용 허위 돌봄일지**: 출석체크가 care_logs를 자동 생성(대표 카드 대상) —
   기존 설계. 포인트와 무관(출석 1회/일 상한)이라 영향 없음.
