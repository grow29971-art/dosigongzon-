# 개발일지 2026-07-15 — 쇼핑 후원 생태계 + 소개 동선 (7/14~15 연속)

## 요약

쇼핑을 "물건 파는 곳"에서 "길집사와 함께 만드는 후원 생태계"로 확장.
사용처 투표 + 후원금 투명 정산 + 1호 프로젝트 배너 + 도시공존 소개 동선을 붙이고,
비로그인 방문자(인스타 유입)가 첫 화면에서 앱을 이해하게 랜딩 최상단을 개편.
후원 문구·하드웨어 설명도 정비. 커밋 다수, SQL 3종 실행.

## 오늘 한 것

### 1. 후원 상품 정리 + 문구 정비
- 후원 전용 상품 3개(겨울나기/쉼터기금/한끼) DB 삭제 + support 카테고리 제거
  (order_items 0건 확인 후 안전 삭제)
- 후원 배너: "수익의 일부분이 길고양이를 위해 쓰입니다 🐱 (사용처 투표 중)"
- 투명성 박스: "서버 유지 등 최소 마진만 남기고 나머지는 전부 아이들에게",
  "모인 금액·쓰인 금액 아래에서 공개", "특정 단체·정당과 무관하게 길집사님들과 함께"

### 2. 사용처 투표 (fund_vote)
- fund_vote_options/fund_votes + RPC(cast_fund_vote 1인1표 upsert,
  fund_vote_results 집계). 개별표 본인만, 합계 공개
- 5개 항목(쉼터/사료/의료비/TNR/겨울나기) + "💡 기타 아이디어" → inquiries로
  관리자 인박스 전송(/admin/inbox, grow29971)
- baseline seed_votes로 시작 표수 표시(실유저 계정 안 건드림). 최종 53명 조정
  (TNR 17 > 구조 13 > 사료 10 > 방한 8 > 쉼터 5). SQL: supabase_fund_vote_*

### 3. 후원금 투명 정산 위젯
- fund_disbursements(관리자 지출 기록, 읽기 공개/쓰기 관리자) +
  /api/shop/fund-settlement(모인=donation합계 / 쓰인=지출합계 / 잔액)
- 3분할 요약 + 최근 사용 내역, 0원이면 "집계 예정"으로 정직하게
- 쇼핑 최상단으로 배치. SQL: supabase_fund_settlement_migration.sql

### 4. 홈 1호 프로젝트 배너 + 돌봄→포인트 안내
- 홈 상단 "🐾 도시공존 1호 프로젝트"(구원→1호, 거창함 제거) 배너 →
  탭하면 /shop 투표. 닫기 가능
- 쇼핑 상단 "매일 돌봄 출석하면 포인트 적립 · 1P=1원" 띠 + 결제창 0P 힌트

### 5. 도시공존 소개 동선 (AboutCityCard)
- 포지셔닝(도시+공존 이름 살림) + 3갈래 현황(앱 서비스중/난로 양산준비/쉼터 개발중)
  + /about 링크. 공용 컴포넌트로 추출
- **비로그인 랜딩(HomeLanding) 최상단** 배치 — 인스타 등 유입 방문자가 첫 화면에서
  앱 이해. 쇼핑에서는 제거 (중복 방지). 히어로 상단 패딩 pt-14→pt-6
- /about 하드웨어 설명 강화: 고양이난로(핫팩 10h vs 난로 약 23h, 핫팩 한계 해결),
  스마트쉼터(동사/원격확인/분산 모니터링 문제해결 중심 재작성)

### 6. 지도 진입 안내 모달
- 지도 첫 진입 시 위치 보호 안내(실좌표 미저장/불규칙 배회/상태 구경),
  localStorage dismiss. "5km" 예시 대신 "동네 안" 표현(실제 3km와 숫자 불일치 회피)
