## 1. 다크모드 가독성 fix

`05ce4e3`에서 1차 시도했는데 여전히 텍스트 안 보인다는 제보.
근본 원인: **Tailwind 4의 `dark:` 변형은 기본적으로 `prefers-color-scheme` 미디어 쿼리** 사용. 프로젝트는 `localStorage` + `html.dark` 클래스 토글 방식이라
지금까지 추가한 모든 `dark:text-...` 클래스가 발동 안 하고 있었음.

`app/globals.css` 최상단에 한 줄 추가:
```css
@custom-variant dark (&:where(.dark, .dark *));
```

이 한 줄이 누락되어서 그동안 누적된 모든 다크모드 변형이 무력화돼 있었음.
RescueBanner, TodayChecklist의 그라디언트 배경도 인라인 `style`에서 Tailwind arbitrary class로 옮김 (인라인은 React 직렬화 차이로 attribute selector 매칭이 불안정).

## 4. 지도 페이지 성능 — 다단계 최적화

### 4-1. 약국 마커 가시 임계 통일
줌아웃 시 약국이 단독으로 노출되던 문제. `app/(main)/map/page.tsx`에서
`isPharm && level >= 7 → return false`. 일반 병원과 동일한 동네 줌(level<7)에서만 표시.

### 4-2. 마커 뷰포트 기반 렌더링 (가장 큰 win)
이전: 134마리 그룹 전체를 매번 DOM 오버레이 생성.
이후: `renderVisibleCats` 함수로 추출 + `idle` 리스너로 화면에 보이는 동만 렌더 + `MAX_CAT_OVERLAYS = 80` 상한 + 학대경보·마릿수 정렬.

### 4-3. sessionStorage 5분 stale-while-revalidate
`MAP_CATS_CACHE_KEY` 모듈 상수 + `invalidateMapCatsCache()` 헬퍼.
- 캐시 hit → 즉시 setCats + 백그라운드 fetchFresh
- 캐시 miss → 평소대로 로드
- 등록·삭제·수정 시 invalidate 호출 (3곳)

### 4-4. 검색어 디바운스 200ms
이전엔 타이핑마다 134개 마커 전체 재생성. `searchQDebounced` 별도 상태로 묶음.

### 4-5. 비크리티컬 fetch 지연
`isCurrentUserAdmin`, `/api/visit` POST/GET을 `requestIdleCallback`으로 첫 페인트 후로 미룸.

### 4-6. 모달 코드 스플리팅
`AddCatModal`, `ReportModal`, `CareLogTab`, `CatLocationPicker` 4개를 `next/dynamic({ ssr: false })`로 분리. 모달 열기 전엔 다운로드 안 함.
