# 개발일지 2026-07-10 — 쇼핑몰 + 토스페이먼츠 결제 (하루 완성)

## 요약

실물 쇼핑몰 기능을 DB부터 결제까지 하루에 구축. 테스트 결제까지 전 구간 검증 완료.
커밋 13개, 전부 프로덕션 배포됨.

## 오늘 한 것

### 1. DB (Supabase)
- `products` / `cart_items` / `orders` / `order_items` 4개 테이블 + RLS
  (`supabase_shop_migration.sql`)
- 카테고리 7종 개편 + 컬럼 확장: badge, is_donation, donation_percent, weight,
  supplier, is_virtual + updated_at 트리거 (`supabase_shop_v2_migration.sql`)
- 시드 상품 21개 — 카테고리별 3개 (`supabase_shop_seed.sql`)
- 본인 pending 주문 삭제 정책 (`supabase_shop_orders_delete_pending_migration.sql`)
- 원자적 재고 차감/복구 RPC — service_role 전용 (`supabase_shop_stock_rpc_migration.sql`)
- 관리자 판별: 기존 `admins` 테이블 + `requireAdmin()` 재사용 (profiles.role 안 만듦)

### 2. 카테고리 (7종)
사료·간식(food) / 모래·위생(sand) / 건강·케어(health) / 장난감·용품(toy) /
급식·쉼터(shelter) / 굿즈(goods) / 후원하기(support — 가상상품, 전액 후원, 배송 없음)

### 3. 페이지
- `/shop` — 후원 배너, 아이콘 칩 필터, 2열 상품 그리드 (뱃지/무료배송/할인율/후원 표기)
- `/shop/[id]` — 서버 컴포넌트 + generateMetadata(SEO/OG), 스와이프 슬라이더,
  후원 안내 카드, 재고 5개 이하 경고, 설명 더보기. 클라이언트 로직은 ProductDetailClient
- `/shop/cart` — 수량 조절, 삭제, 배송비 합계
- `/shop/checkout` — 배송지 입력(다음 우편번호), 주문 메모, 결제창 호출
- `/shop/orders`, `/shop/orders/[id]` — 주문 내역, 상태 타임라인, paid 주문 취소
- `/shop/payment/success|fail` — 승인 요청/실패 안내
- `/admin/products`, `/admin/orders` — 상품 CRUD(이미지 5장, cat-photos 버킷 product_ prefix),
  주문 상태·운송장 관리, 취소 시 재고 복구

### 4. 결제 (토스페이먼츠)
- **결제창 방식** (`toss.payment().requestPayment({method: "CARD"})`) —
  결제위젯이 아님! 위젯 키(test_gck_)는 전자결제 이용 신청(사업자 필요)을 해야 발급이라,
  API 개별 연동 키(test_ck_)로 쓸 수 있는 결제창으로 전환함
- `/api/payment/confirm` — 금액 위변조 검증 → 주문 선점(경합 방지) →
  선(先) 재고 확보(원자적 RPC) → 토스 승인 → 확정. 실패 시 재고 원복.
  ALREADY_PROCESSED_PAYMENT는 성공 처리(타임아웃 재시도 대응)
- `/api/payment/cancel` — pending 즉시 취소 / paid는 토스 환불 후 취소 + 재고 복구
- 키: `NEXT_PUBLIC_TOSS_CLIENT_KEY` / `TOSS_SECRET_KEY` (.env.local + Vercel, 테스트 키)

### 5. 삽질 기록 (다음에 참고)
- **하단 버튼 실종**: BottomNav(z-50)가 상세/장바구니/주문서의 고정 버튼 바(z-40)를
  완전히 덮고 있었음 → `/shop/*` 하위 경로에서 네비 숨김으로 해결
- **주소 검색 차단**: 다음 우편번호 iframe이 `postcode.map.daum.net`이 아니라
  **`postcode.map.kakao.com`**에서 뜸(카카오 도메인 이전). postcode.v2.js를 직접
  받아서 URL 확인 후 CSP frame-src에 추가
- **결제위젯 로드 실패**: test_ck_ 키는 widgets()에 못 씀. 계정에 위젯 키가 없으면
  결제창 방식이 정답
- CSP 추가된 도메인: js.tosspayments.com(script), *.tosspayments.com(connect/frame),
  pay.toss.im(frame), postcode.map.kakao.com(frame)
