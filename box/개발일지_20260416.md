## 2. 전국 동물병원 공공데이터 동기화 (14:10~16:00)

### 2-1. 시스템 구조
- **API 라우트**: `POST /api/admin/sync-hospitals` (관리자 전용)
- **데이터 소스**: 카카오 키워드 검색 API (`동물병원` rect 검색)
- **저장**: `rescue_hospitals` 테이블 (source='kakao')
- **환경변수**: `KAKAO_REST_API_KEY` (카카오 개발자 콘솔 REST API 키)

### 2-2. 검색 알고리즘
- 한반도를 그리드로 분할 후 각 셀에서 "동물병원" 키워드 검색
- **밀집 지역** (서울/경기/인천/부산/대구/대전/광주): **0.05° × 0.05°** 미세 그리드
  - 서울은 한 셀당 최대 30~45개 → 분할 불필요, 바로 수집
- **나머지 전국**: 0.5° × 0.7° 큰 그리드 + 필요시 4분할 재귀
- 결과가 pageable_count(45) 초과 시 2개씩 병렬 분할

### 2-3. Rate Limit 대응 (핵심 시행착오)
- **문제**: 카카오 API pageable_count = 45 (한 쿼리당 최대 45건)
- **1차 시도**: 큰 그리드(0.8°) + Promise.all 4분할 → 서울 전체 skip (rate limit 429)
- **2차 시도**: 쓰로틀(동시 5개) + 재시도 3회 → 여전히 서울 누락
- **3차 (최종)**: 밀집 지역 미세 그리드 분리 → 분할 자체가 불필요 → 서울 1069개 정상 수집
- **추가 발견**: Supabase `.select("*")` 기본 1000행 제한 → `range()` 페이지네이션으로 전체 로드

### 2-4. DB 스키마 변경
```sql
-- rescue_hospitals 테이블 추가 컬럼
source TEXT DEFAULT 'manual'     -- 데이터 출처 (manual/kakao)
kakao_place_id TEXT              -- 카카오 장소 ID (유니크)
hidden BOOLEAN DEFAULT false     -- 폐업 신고 숨김
```

### 2-5. 최종 결과
- 전체 **4,129개** 병원 (kakao 3,852 + manual 277)
- 서울 **1,069개**, 경기 **560개**, 인천 **52개** 등
