## 1. 이전 Expo 앱 마이그레이션 시도 (오후 1시~5시)

### 진행 내용
- `box/zz` (이후 `box/aa`)에 있던 기존 Expo React Native 앱을 Next.js 16.2.2로 마이그레이션 시도
- 타입 시스템, AuthContext, DataContext, 로그인/회원가입, 커뮤니티, 채팅, 마이페이지, 보호지침, 스토어, 도움센터 등 전체 페이지 변환
- 총 ~10,500줄 규모의 대규모 마이그레이션

### 발생한 문제들
1. **`nul` 파일 문제** — `aa/my-first-app/nul` (Windows 예약 파일명)이 Turbopack 크래시 유발
2. **Google Fonts 무한 대기** — `Geist` 폰트 다운로드가 네트워크에서 블록되어 SSR 무한 로딩
3. **Turbopack root 오류** — 상위 폴더(`C:\Users\acer\`)의 `package-lock.json`과 충돌하여 잘못된 root 설정
4. **SSR 하이드레이션 불일치** — localStorage 접근과 시간 기반 함수(`getGreeting`)가 서버/클라이언트 HTML 불일치 유발
5. **Provider hydration 실패** — `dynamic` import + `ssr: false`로 Context Provider 감쌌으나 정상 동작하지 않음

### 결론
- 브라우저에서 무한 로딩이 반복되어 **전체 리셋 결정**
- `git checkout d632a0b -- .` 으로 초기 커밋 상태로 복원
- `box/zz`, `box/aa` 폴더 모두 삭제

## 5. 교훈 & 메모

- **Next.js 16.2.2 + Turbopack**: 상위 폴더에 `package-lock.json`이 있으면 root 충돌 발생 → `next.config.ts`에서 절대경로 지정 필수
- **Google Fonts**: 네트워크 환경에 따라 SSR을 완전히 블록할 수 있음 → 시스템 폰트로 시작하고 나중에 추가
- **SSR + localStorage**: `if (!mounted) return null` 패턴으로 클라이언트 전용 렌더링 강제
- **대규모 마이그레이션보다 처음부터 만드는 게 더 빠를 수 있음** — 특히 프레임워크가 완전히 다를 때 (React Native → Next.js)
