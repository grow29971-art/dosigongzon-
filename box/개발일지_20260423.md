## 3. 비밀번호 재설정 흐름 완성

**문제 발견:**
- `/find-account`에 비밀번호 찾기 탭은 이미 있었으나, 메일 링크 redirectTo가 `/login`이라 실제로는 작동 안 함
- 로그인 페이지에 새 비밀번호 입력 폼이 없어 메일 링크 클릭해도 재설정 불가능했음

**조치:**
- `/reset-password` 페이지 신규 생성
  - 세션 확인 → 새 비밀번호 입력 폼 + 강도 표시 → `auth.updateUser({ password })` → 자동 로그인
  - 만료/세션 없음/완료 3가지 상태 분기
- `/find-account` redirectTo를 `/api/auth/callback?next=/reset-password`로 변경
  - PKCE code 흐름이 callback에서 세션 교환 → reset-password로 이동
- 로그인 페이지 "아이디 · 비밀번호 찾기" 링크를 primary 색 + underline로 강조 (잘 안 보였음)

## 4. 사이트 보안 점검

어제(4/22) 코드 레벨 보안은 거의 다 정비됨. 오늘은 누락 보강 + 검증.

### 4-1. npm audit
- 3 moderate 취약점 (resend → svix → uuid 체인)
- uuid bounds check는 buf 인자 직접 사용 시에만 트리거 — resend는 그렇게 쓰지 않음 → 실제 노출 거의 0
- 다음 정기 업데이트 시 같이 처리

### 4-2. RLS 정책 전수 점검
- 28개 public 테이블 전부 RLS 활성화 ✅
- 정책 누락 없음. service_role 전용 테이블(admins, anon_visit_dedupe)은 의도적으로 정책 없음
- 처음 grep이 한 줄짜리만 잡아서 누락된 것처럼 보였으나 multiline으로 재확인 → 모두 정상

### 4-3. API rate limit 보강
- `app/api/auth/log-error/route.ts` — IP당 분당 20회 (봇 폭격으로 auth_error_logs 누수 방지)
- `app/api/weather/route.ts` — IP당 분당 10회 (OpenWeatherMap 무료 quota 보호)
- 기존 보호: chat(레벨별), hospitals/report-closed(1시간 5회), visit/check-nickname/find-email/social-proof

### 4-4. CSP + 보안 헤더
- 어제 작업으로 이미 완비 (CSP directives + 8개 보안 헤더 + API CORS) → 추가 작업 불필요

## 6. 이미지 최적화 — next/image 전환

10개 파일에서 `<img>` / `background-image: url()` 사용 중. 핵심 공개 페이지 8개를 next/image로 전환.

**전환된 페이지:**
- `app/areas/[slug]/page.tsx` (고양이 카드)
- `app/areas/[slug]/[dong]/page.tsx` (고양이 카드)
- `app/(main)/cats/[id]/page.tsx` (커버 + 갤러리 썸네일)
- `app/(main)/community/[id]/page.tsx` (게시글 첨부 + 작성자 아바타)
- `app/(main)/community/category/[cat]/page.tsx` (썸네일 + 아바타)
- `app/(main)/mypage/page.tsx` (프로필 + 본인 고양이)
- `app/components/MyActivityDashboard.tsx`
- `app/components/HomeAuthed.tsx` (홈 캐러셀)

**Vercel Image Optimization 이슈:**
- 첫 배포 후 모든 `/_next/image` 요청이 `INVALID_IMAGE_OPTIMIZE_REQUEST` 400 반환 → 이미지 전체 깨짐
- 원인 추정: Vercel Image Optimization quota 또는 플랜 제한
- **임시 fix:** `next.config.ts`에 `images.unoptimized: true` 적용
- 효과: lazy loading·sizes 등 next/image 기본 기능은 유지, 자동 WebP/AVIF 변환만 비활성
- 향후 Vercel 대시보드에서 Image Optimization 활성화 확인 후 `unoptimized: false` + `images.unsplash.com` 추가 (더미 데이터 호스트)

**남은 작업 (트래픽 적은 곳, 다음에):**
- admin/* 페이지들
- AddCatModal, CareLogTab, community/write 미리보기 (blob URL이라 next/image 적용 어려움)
- messages, map, pharmacy-guide
