## 3. Sentry 에러 모니터링 적용 (커밋 `491469e`)

### 진단
`sentry.server.config.ts`는 있고 `instrumentation.ts`에서 onRequestError 자동 주입됨 (unhandled 에러는 캡처). 그러나 **`try-catch`로 잡힌 에러는 framework가 "handled"로 보고 캡처 안 함**. console.error만 남고 프로덕션 모니터링 불가.

### 헬퍼 도입
신규 `lib/error-report.ts`:
```ts
export function reportError(scope: string, err: unknown, extra?: Record<string, unknown>) {
  console.error(`[${scope}]`, err, extra ?? "");
  Sentry.captureException(err, { tags: { scope }, extra });
}
```
`scope` 태그로 대시보드에서 라우트별 필터링 가능 (`scope:cron/news-crawl` 등).

### 적용 (11개 라우트)
- 외부 API: `weather`, `turnstile-verify`, `chat`(Gemini), `models`
- Cron: `sync-pharmacies`, `news-crawl`, `import-shorts`, `cleanup-area-chats`, `cleanup-read-dms`
- 그 외: `event/keyring/admin-notify`, `admin/insights`

### 노이즈 방지 결정
- Gemini 429 / YouTube quota 초과는 예측되는 상황 → `console.warn`만, Sentry 제외
- Push subscription 만료(410/404)는 정상 흐름 inner catch라 그대로 유지

## 5. Kakao Maps 타입 정의 (커밋 `1f6d171`)

### 문제
3개 파일에 `declare global { interface Window { kakao: any } }` 중복 + 콜백 5건이 `(result: any, status: any)` 패턴. 공식 SDK가 타입 미제공.

### 해결
신규 `lib/kakao-types.ts` — 실제 사용 API만 최소 타입화:
- `KakaoMap`, `KakaoLatLng`, `KakaoMapMouseEvent`
- `KakaoGeocoder` (coord2RegionCode, addressSearch)
- `KakaoPlaces` (keywordSearch)
- `KakaoCustomOverlay`, `KakaoCircle` (with `setMap`)
- `KakaoMapsServices.Status`

Window 글로벌 선언을 1곳으로 통합, 5개 파일에서 콜백 파라미터 타입 자동 추론. `useRef<KakaoMap | null>(null)`처럼 ref 타입도 명시. `mapInstanceRef.current` null guard 추가.

### 정책
새 Kakao API 사용 시 `lib/kakao-types.ts`에 한 줄씩 추가. 전체 SDK 미러링하지 않음 — 사용분만.

## 7. Storage diet cron 자동화 (커밋 `1e8259d`)

### 배경
`box/supabase_storage_diet_cleanup.sql` PART 2 정리(push_alert_log 30일, auth_error_logs 90일, anon_visit_dedupe 30일)가 수동 실행으로만 유지되어 누적 부담 잠재.

### 신규 cron
`/api/cron/storage-diet`:
- 매일 17:00 UTC (= 02:00 KST 다음날) 실행 — 한가한 시간대
- 세 테이블 순회, 개별 실패해도 다른 건 계속
- 각 실패는 `reportError("cron/storage-diet/<table>", err)`로 Sentry 전송
- 응답: `{ ok, totalDeleted, results: { table: { deleted, error } }, at }`

### 자동화 제외
PART 4 고아 storage 파일 정리는 race condition 리스크(업로드 중 파일이 "고아"로 보일 수 있음)로 수동 유지. SQL 파일에 자동화 사실 주석 추가.
