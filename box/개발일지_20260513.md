## 🔴 critical fix — 회원가입 봇방지 widget 안 보이던 문제

### 증상

`/signup` 페이지에서 "봇 방지 확인하세요" 안내만 보이고 Turnstile widget 자체가 안 떴음. 신규 가입이 사실상 막혀있었던 상태.

### 진단 도구

`box/debug_turnstile.mjs` puppeteer 스크립트로 headless 진단. window.turnstile 존재 여부, iframe 개수, console 에러, container DOM 상태를 한 번에 출력.

### 원인 2개 (둘 다 잡아야 동작)

1. **sitekey trailing 개행**: `NEXT_PUBLIC_TURNSTILE_SITE_KEY` 값 끝에 literal `\n` 포함. Cloudflare가 `Invalid input for parameter "sitekey"` 반환. `.env.local`을 그대로 Vercel env에 옮겼는데 `"..."` 안의 `\n` 문자가 escape 안 되고 그대로 들어감.
2. **hostname 미등록**: Cloudflare Turnstile 대시보드의 해당 widget에 `dosigongzon.com` hostname이 허용 목록에 없었음. SDK 로드는 되지만 `Error: 110200 (Invalid domain)`으로 silent fail.

### fix

- `app/components/TurnstileWidget.tsx`:
  - `siteKey = process.env.NEXT_PUBLIC_TURNSTILE_SITE_KEY?.trim()`
  - `appearance: "always"` + `size: "normal"` 명시 (managed widget의 자동 invisible 모드 회피)
  - `render()` 실패 시 `console.error`로 silent fail 방지
- Cloudflare 대시보드: Hostname management에 `dosigongzon.com` 추가 (사용자 수동)

같은 원리로 다른 `NEXT_PUBLIC_*` 값(`NEXT_PUBLIC_SUPABASE_*` 등)에도 trailing `\n` 있음. Supabase는 우연히 클라이언트 라이브러리가 trim하지만 Turnstile은 raw로 전달돼 노출됐음. 다음 외부 API 연동 시 `.trim()` 습관화 필요.

관련 커밋: `ddfba20`, `292cad9`

## ⚡ 가장 큰 영향 — Storage 사진 백필

### 발견

Lighthouse 측정에서 LCP 보틀넥 추적 → 일부 cat 사진이 **2-10MB** 짜리 원본 그대로 다운로드. 추적 결과 `seed_real_dummy_cats.mjs` Node 시드 스크립트가 클라 압축 파이프라인(`convertImageToWebp`)을 우회해서 dummy-real 폴더에 원본 JPEG을 그대로 업로드했었음.

### 도구

`box/compress_dummy_photos.mjs`: 일회성 백필 스크립트

## 🔧 Supabase Image Transformation 활성화

Pro 플랜이라도 Image Transformation은 Dashboard에서 **별도 토글**. 이전 핫픽스(`ce1b855`)로 두 곳 상수(`SUPABASE_IMAGE_TRANSFORM_ENABLED` / `TRANSFORM_ENABLED`)를 `false`로 두고 원본 URL fallback 상태였음.

오늘 사용자가 Dashboard에서 Enable image transformation 토글 ON 후 두 상수 모두 `true`로 복원(`45be9d1`).

활성 후 효과: 이미지 386KB → 39KB (-90%). 모든 마커·상세·라이트박스·댓글 사진이 `/render/image/public/?width=...`로 변환된 사이즈만 다운로드됨.

활성화 위치 + 동작 확인 명령은 메모리 `project_dosigongzon_supabase_pro.md`에 기록.

## 🚀 Performance — LCP 보틀넥 사냥

### Style & Layout 2128ms — `content-visibility: auto`

Lighthouse mainthread-work-breakdown에서 **Style & Layout이 2128ms** 점유 발견(랜딩 13개 섹션 + 풍부한 인라인 스타일). `app/globals.css`에 `.cv-auto { content-visibility: auto; contain-intrinsic-size: auto 600px; }` 유틸 추가하고 below-fold 섹션 20개에 적용:

- HomeLanding 9개 (최근 고양이~만든 사람)
- HomeAuthed 4개 (돌봄왕, 인기고양이, 인기커뮤니티, 사회소식)
- areas/[slug] 3개, areas/page.tsx 2개, regions/[sido] 2개

결과: Style & Layout **2128ms → 1426ms (-33%)**. Chrome/Edge/Safari 18+ 지원, 미지원 브라우저는 정상 렌더 폴백.

### 추가 LCP / 번들 fix

- **Sentry Replay 지연 로드**: `replayIntegration`이 모든 페이지 첫 페인트 번들에 포함돼 ~70KB. `beforeSend`에서 `lazyLoadIntegration`으로 동적 import. 평소 녹화는 어차피 0%라 손해 없음.
- **Service Worker v5→v6**: `/_next/static/*`는 content-hash immutable → cache-first 전환. 2nd+ 페이지뷰 LCP·FCP 즉시 단축.
- **HomeAuthed PushOptInCard** lazy 분리 (페이지 끝, viewport 밖).
- **TS target ES2017 → ES2020**: legacy JS polyfill 절감. PWA + 모바일 위주라 안전.
- **Storage cacheControl 1h → 30d**: 8개 업로드 경로 + 기존 200+ 파일 cacheControl 갱신. 반복 방문자 LCP 즉시.
