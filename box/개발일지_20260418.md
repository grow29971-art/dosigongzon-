## 1. 활동 지역 데이터 모델

### 테이블 `user_activity_regions`
| 컬럼 | 타입 | 설명 |
|------|------|------|
| user_id | UUID | auth.users FK, ON DELETE CASCADE |
| slot | SMALLINT | 1 또는 2 |
| name | TEXT | 지역 이름 (예: 구월동) |
| lat / lng | DOUBLE PRECISION | 중심 좌표 |
| radius_m | INTEGER | 300~5000m 체크 |
| is_primary | BOOLEAN | 주 활동 지역 플래그 |
| created_at / updated_at | TIMESTAMPTZ | 타임스탬프 |

- **PK**: `(user_id, slot)` — 유저당 최대 2개
- **유니크 부분 인덱스**: `WHERE is_primary = true` — 주 활동 지역은 한 개만
- **RLS**: 본인만 read/write, 정지 유저 INSERT 차단
- **Trigger**: `updated_at` 자동 갱신

### 반경 프리셋
500m / 1km / 2km / 3km / 5km — 당근마켓 UX 참고

### 파일
- `box/supabase_activity_regions_migration.sql`

## 3. 활동 지역 설정 페이지

### `/mypage/activity-regions`
- **슬롯 탭**: 활동 지역 1 / 활동 지역 2 — 각각 색상(갈색/파랑)
- **Kakao 지도 + 원**: 중심 마커 + Circle 오버레이 실시간 반영
- **지도 터치**: 중심점 이동 + 역지오코딩으로 동 이름 자동 입력
- **주소 검색바**: Kakao Places `keywordSearch` 로 장소 검색 → 결과 클릭 시 이동
- **내 위치 FAB**: GPS로 한 방에 이동
- **반경 칩**: 500m/1/2/3/5km 토글
- **주 활동 지역**: 별표 토글 (홈/지도 기본 기준)
- **다른 슬롯 표시**: 활성 아닌 슬롯은 점선 원으로 희미하게 참고 표시

### 마이페이지 진입점
- "지원" 섹션 위에 **"내 설정 → 활동 지역 설정"** 카드 추가 (갈색 MapPin)

## 4. 지도 페이지 통합

### 활동 지역 탭 (헤더 아래)
- 🌍 **전체** / 📍 **지역1** / 📍 **지역2** / ⚙ 지역 설정
- **고양이는 반경 밖도 그대로 표시** — 학대 경보 등 놓치지 않도록
- 탭 클릭 시 해당 지역 중심으로 카메라 이동 + Circle 강조
- 비활성 지역은 점선 원으로 옅게 표시
- 설정 없으면 → "📍 내 활동 지역 추가하기" CTA 한 줄

### Circle 오버레이
- 활성 지역: 실선 2px, fill 8%
- 비활성 지역: 점선 1px, fill 2% (존재는 보임)
- 이름 라벨 CustomOverlay 중앙 부착

### 초기 중심 우선순위
GPS > 주 활동 지역 > 기본 중심(인천 남동구청)

### 내 위치 버튼 로직 변경
- 현재 필터가 slot이면 → 해당 지역 중심
- 그 외 → 주 활동 지역 → GPS 순 폴백

## 8. 로그인 실패 진단 + 친절한 안내 + 폴백

소셜 로그인에서 왜 실패하는지 알 수 없으니 추측만 하게 되던 문제. 3단계로 해결.

### (1) 에러 수집 — `auth_error_logs` 테이블
| 컬럼 | 설명 |
|------|------|
| provider | google / kakao / email / magic_link |
| stage | redirect / callback / exchange / client |
| error_code | KOE205, redirect_uri_mismatch 등 |
| error_desc | error_description 원문 |
| user_agent, url, referrer, created_at | 환경 정보 |

- RLS: 삽입은 누구나, 읽기·삭제는 관리자만
- 파일: `box/supabase_auth_error_logs_migration.sql`

### (2) 로깅 포인트
- `/api/auth/callback` — OAuth provider가 `error=` 돌려주면 DB에 저장 후 파라미터 포함해서 `/login`으로 리다이렉트
- `/api/auth/callback` — `exchangeCodeForSession` 실패해도 로깅
- `/api/auth/log-error` — 클라이언트에서 `signInWithOAuth` 자체가 실패한 경우
- `app/login/page.tsx` — 에러 파라미터 들고 돌아왔을 때 redirected 단계로 한 번 더

### (3) 에러 매퍼 — `lib/auth-errors.ts`
- 10+ 패턴 (KOE205/KOE006/redirect_uri_mismatch/access_denied/cookie/email_not_confirmed 등)
- 각 에러마다 `title / body / tip / severity` 반환 → 로그인 페이지에서 색상·아이콘 다르게 표시

### (4) 로그인 페이지 가이드 카드
- `danger` (빨강) / `warn` (주황) / `info` (파랑) 3단계 색상
- 에러 코드를 하단에 monospace로 노출 (유저가 스크린샷 보낼 때 유용)

### (5) 매직링크 폴백
- "소셜 로그인이 안 되면 이메일로 로그인 링크 받기" 버튼
- `supabase.auth.signInWithOtp({ email })` — 비밀번호 없이 이메일 링크만으로 로그인
- 가장 호환성 좋은 옵션 (쿠키 차단/인앱브라우저/PWA 모두 대응 가능)
- 발송 성공 시 안내 카드 + "다른 이메일로 다시 보내기" 버튼
