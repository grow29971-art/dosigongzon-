### 6. Meta Pixel race condition·silent drop 방어 (6782d04)
**문제**: 어제 광고 캠페인 결과에서 가입 전환 미카운팅 의심. 원인:
1. Lead 이벤트가 OAuth redirect 직전 발사 → 비콘 송신 전 페이지 unload → 이벤트 로스
2. ConsentManager가 fbq 스크립트 로드 중 호출되면 silent drop
3. trackPixelOnce가 dedup 마킹을 발사 전에 해서 timeout 시 영구 미발사

**`lib/meta-pixel.ts`**:
- `whenFbqReady(maxWaitMs=2000)` — fbq 로드까지 polling 대기 헬퍼
- `trackPixelEventAsync` — 대기 + 250ms grace로 비콘 송신 완료 보장
- `trackPixelOnce` dedup 마킹을 fbq 발사 성공 후로 이동 — timeout 시 재방문에 재시도

**`app/signup/page.tsx:119`**: `trackPixelEvent("Lead", ...)` → `await trackPixelEventAsync("Lead", ...)`로 변경 — OAuth redirect 전 비콘 송신 보장.

**미해결 (D-4 freeze)**: 쿠키 동의 거부 유저의 추적 누락 → 서버사이드 Conversions API(CAPI) 필요. 마감 후로 보류.
