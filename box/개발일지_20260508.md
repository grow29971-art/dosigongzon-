# 개발일지 — 2026-05-08 (금)

비공개 테스트 운영 중 들어온 테스터 피드백 두 건 핫픽스. 어드민 숏츠 페이지에서 묵혀둔 정리 작업도 같이.

## 한 줄 요약
**테스터 피드백 핫픽스**: 숏츠 뒤로가기 시 소리 켜지는 버그 + 갤럭시 폴드 펼친 화면에서 지도 양옆 빈 공간 → 둘 다 해결 후 프로덕션 배포.

## 1. 숏츠 뒤로가기 시 소리 켜짐 버그
테스터 제보:
> 도시공존 영상에서 음소거 상태에서 뒤로가기 누르면 소리가 켜진 채로 뒤로 가요.

### 원인
`ShortsFeed.tsx`의 자동 unmute 리스너:
- 첫 사용자 제스처(클릭/스크롤) 발생 시 `setMuted(false)` → 모든 iframe에 `unMute` postMessage
- 음소거 토글 버튼은 `data-mute-toggle` 속성으로 제외했지만 **뒤로가기 `<Link>`는 제외 안 됨**
- 뒤로가기 탭 → 자동 unmute 발동 → iframe unMute → 네비게이션 직전에 잠깐 소리 켜짐

### 해결
- 데이터 속성 이름을 의미 명확하게 `data-no-unmute`로 일반화
- 뒤로가기 `<Link>`와 음소거 토글 `<button>` 둘 다 같은 속성 부여
- enable() 핸들러에서 `[data-no-unmute]`로 제외 체크

### 교훈
"unmute 제외 대상"이 늘어날 가능성이 있을 땐 처음부터 일반 속성명으로 시작. `data-mute-toggle`처럼 한 가지 용도에 묶인 이름은 두 번째 케이스 만나면 다 고쳐야 함.

## 2. 갤럭시 폴드 펼친 상태 지도 양옆 빈 공간
테스터 제보:
> 폴드 펼치면 지도가 꽉 안 차고 양 옆이 비어요.

### 원인
`(main)/layout.tsx`:
```tsx
<main className="pb-20 mx-auto w-full max-w-lg">{children}</main>
```
- `max-w-lg` (32rem = 512px) 제약 → 모바일 폭으로 컨텐츠 박제
- 폴드 펼친 화면(>1800px) 또는 일반 태블릿에서는 양옆 회색 공간 노출
- 이 제약은 일반 페이지(community/mypage 등)에는 의도된 동작이지만 **지도만큼은 풀스크린이어야 함**

### 해결
지도 컨테이너에 CSS breakout 패턴:
```tsx
style={{
  width: "100vw",
  marginLeft: "calc(50% - 50vw)",
}}
```
- 부모 `max-w-lg`를 무시하고 viewport 전체 폭 차지
- 모바일에서는 어차피 `100vw == max-w-lg` 이므로 변화 없음
- 폴드/태블릿에서만 풀스크린으로 펼쳐짐

### 교훈
모바일 우선 레이아웃에서 `max-width` 제약을 두는 건 일반 콘텐츠엔 맞지만 **지도/카메라/캔버스 같은 풀스크린 인터랙션 요소는 항상 예외**. 처음부터 페이지별 레이아웃 분리 고려할 것.

## 3. 어드민 숏츠 부수 정리 (이전 작업분)
오늘 핫픽스와 별개로 묵혀있던 변경:

### 3-1. PostgREST 1000개 limit
`listAllShorts()`가 sorting 후 `.range(0, 9999)` 명시 안 해서 어드민 페이지에 999개까지만 떴음. 숏츠 1000개 넘어가면 잘림.
→ `.range(0, 9999)` 추가.

### 3-2. ShortsAdminStats 제거
어드민 페이지의 통계 표시 블록을 떼냄 (모듈 + 호출 + 상태 다 제거).

### 3-3. 숏츠 임포트 키워드 풀 확장
`/api/cron/import-shorts/route.ts` 검색어 풀:
- 기존: 길고양이 중심 ~30개
- 확장: 고양이/강아지/소동물/야생/조류/농장/수생/파충류 × 감정(귀여운/웃긴/슬픈/밈) 조합 200개+
- 도시공존 컨셉상 동물 일반으로 폭 넓힘 (메인은 여전히 고양이 비중)
