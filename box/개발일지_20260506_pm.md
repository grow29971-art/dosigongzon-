## 2. Play Store 첫 출시 시도 — 대장정

### 2-1. AAB 빌드 (PWABuilder.com)
오전에 만든 `kr.dosigongzon.app` 패키지명으로 새 AAB 빌드 필요.
로컬에 Java/Android SDK 없어서 PWABuilder.com 웹 도구 사용 → 가입·설치 없이 5분.

### 2-2. 패키지명 충돌 (1차)
`com.dosigongzon.app` 으로 처음 시도 → "이미 사용 중인 패키지 이름" 거부.
- 누가 선점한 듯. `kr.dosigongzon.app`로 변경 (오전에 처리).
- assetlinks.json도 사이트에 갱신·재배포.

### 2-3. PWABuilder Package ID 입력 함정
다이얼로그에서 Package ID 기본값이 들어있어 실수로 그대로 둠 → 잘못된 패키지명 AAB 빌드 → Play Console에서 또 거부.
**교훈**: PWABuilder 다이얼로그 매번 Package ID 칸 확인 필수.

### 2-4. 서명 키 mismatch
PWABuilder가 매 빌드마다 새 keystore 생성 → 1차 업로드한 AAB의 키를 Play Console이 "upload key"로 등록 → 2차 빌드의 다른 키로 업로드 시도 → "잘못된 키로 서명" 거부.

> Expected SHA-1: 76:40:1C:60... (1차 업로드 키)
> Got SHA-1: 52:4A:14:BF... (2차 빌드 키)

해결: 첫 PWABuilder ZIP의 `signing.keystore` + `signing-key-info.txt`(비밀번호) 그대로 재사용 → "Use mine" 옵션으로 빌드.

### 2-5. 버전 코드 충돌
실수로 같은 화면 닫고 재업로드 → 버전 코드 1이 이미 사용됨 거부.
PWABuilder에서 **Version code: 2** 로 다시 빌드.

### 2-6. 업그레이드 호환성 차단
프로덕션 출시 시도하니 "기존 사용자가 새 AAB로 업그레이드 못 함" 차단.
원인: 내부 테스트에 v1·v2 등 여러 버전이 누적된 상태에서 v2 → v2 promote 충돌.
**Version code: 3** 로 또 다시 빌드 → 이번엔 통과.

### 2-7. 마침내 내부 테스트 출시 성공
"내부 테스터에게 제공됨" 상태 도달. 5~30분 내 테스터 설치 가능.

### 2-8. 프로덕션 가는 길은 막힘
Google 2023년 11월 신규 개인 개발자 정책:
- 프로덕션 출시 전 **비공개 테스트 트랙** 필수
- **최소 12명 테스터** + **14일간 옵트인 유지**
- 그 후 "프로덕션 액세스 신청" 가능

→ 오늘 일반 Play Store 등재는 불가능. 5/20 이후 신청 가능.

## 3. 발견한 함정 정리
1. **PWABuilder Package ID 다이얼로그 함정**: 매 빌드마다 다이얼로그 떴을 때 Package ID 칸 다시 확인. 기본값으로 리셋되는 경우 있음.
2. **서명 키 일관성**: 첫 빌드의 keystore 파일 + 비밀번호 메모장에 안전 보관 필수. 잃으면 upload key reset 신청해야 하는데 24~48시간 소요.
3. **버전 코드는 단조 증가**: 한 번 사용된 버전 코드는 다시 못 씀. 폐기해도 마찬가지.
4. **Google 신규 개발자 정책**: 프로덕션 가려면 closed testing 14일+12명. 오늘 첫 출시한 신규 개발자에겐 큰 장벽.
5. **테스터는 Google 계정만 가능**: 네이버·카카오 메일 안 됨.

## 가져갈 교훈
- **신규 개발자는 첫 출시 일정에 +14일 ~ 1개월 더 잡아야 함** — Google 정책상.
- **TWA 빌드는 PWABuilder.com이 압도적으로 편함** — 로컬 Java/Android SDK 셋업 불필요.
- **PWABuilder 빌드 흐름 핵심**: Package ID + Signing key (Use mine) + Version code 세 가지가 매번 정확해야 함. 한 번 실수로 하루 날림.
- **Play Console UI는 신규 개발자에게 친화적이지 않음** — 메뉴 위치 자주 바뀌고, 정책 변경에 따른 차단이 많음.
- **사용자 가이드 모달은 dismiss 기간 30일 정도가 적절**. 너무 짧으면 (7일) 자주 보여 짜증, 너무 길면 (영구) 새 기능 못 알림.
