## 21. 사후 코드 검토 + 버그 2건 수정

오늘 저녁에 쌓인 변경을 Explore 에이전트로 감사.

### 21-1. 실제 수정한 것

**`lib/dm-repo.ts` — 타입 캐스팅 오도**
- `getConversations`에서 SELECT 컬럼 축소했지만 `data as DirectMessage[]`로 전체 타입 캐스팅
- 없는 `photo_url` 같은 필드가 타입에 존재하는 듯 보이는 위생 문제
- `Pick<DirectMessage, ...>`로 엄밀히 제한

**`lib/care-logs-repo.ts` — 로케일 파싱 견고화**
- `toLocaleString("en-GB", { ... })` 후 `"DD/MM/YYYY, HH:mm"` 포맷 가정해 split
- 일부 런타임/브라우저에서 padding/구분자가 다르게 나올 여지
- `Intl.DateTimeFormat(...).formatToParts()` 기반으로 교체 — type 필드(year/month/hour)로 명시적 접근

### 21-2. 버그 아님으로 판정

- **SECURITY DEFINER 안의 `auth.uid()`** — Supabase는 JWT GUC(`request.jwt.claim.sub`)로 auth.uid()를 구현. SECURITY DEFINER는 role만 바꾸지 GUC는 유지 → PostgREST 컨텍스트에서 정상 작동
- **longestStreak `new Date("YYYY-MM-DD" + "T00:00:00")` 차이 계산** — prev/curr 동일 해석 규칙이고 KST는 DST 없음 → 연속 일자 간 정확히 86400000ms
- **streak-reminder KST 경계** — 21:30 KST 실행이라 자정 경계 이슈 없음
