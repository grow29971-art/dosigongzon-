## 1. 위급 고양이 자동 알림 cron

`/api/cron/health-alert-push` 신규.
- 매일 06:00 UTC = 15:00 KST
- 로직: `health_status caution|danger` + 마지막 돌봄 3일 초과 → 같은 동(`region` 매칭) 활동 유저에게 푸시
- 안전장치: 본인 caretaker 제외, marketing_push_enabled=true만, 1회 실행 max 30 cats × 20 users
- 메시지: `🐾 위험 상태 ○○ — XX동의 ○○이가 N일째 안부가 없어요`
- 기존 `webpush` + `push_subscriptions` + `user_activity_regions` + `care_logs` 인프라 100% 재활용

### 1-2. push_alert_log dedup
첫 배포 후 발견한 P2 버그: 같은 (유저, 고양이) 페어로 매일 푸시 → 스팸.
`box/supabase_push_alert_log_migration.sql` 신규 테이블 + cron에 dedup 로직 추가.
- 24h 내 같은 페어 푸시 받았으면 스킵
- 성공한 푸시만 INSERT (실패는 다음 cron에서 재시도)
- 로그에 `skipped` 카운트 포함 (admin 통계용)

### 1-3. 알림 센터(in-app) 짝
`getNotifications()`에 `urgent_in_area` 타입 추가. 푸시 못 받는 환경에서도 앱 진입 시 위급 알림 확인 가능.
초기 버그: `createdAt`을 last_care 시점으로 잡아서 매일 재정렬되어 새 알림처럼 보임 → 위급 진입 시점(last_care + 3일)으로 고정해서 재발 방지.
헤더 종 배지 카운트도 합산.

## 4. 키링 응모 폼 완전 단순화

어제 1차로 4필드 → 1필드(고양이 이름+사진), 오늘은 **버튼 1개**까지.
- 페이지: 폼 전부 제거, 응모하기 버튼만

- admin 알림: "○○ 님이 키링 추첨에 응모했어요" 한 줄
- SQL 업데이트: name·address·phone·cat_photo_url 모두 NOT NULL 제거

### 4-2. admin/event-keyring NULL 안전

- 응모자 닉네임 별도 fetch (profiles)

- 단순 응모 카드에 "사진·연락처 미수집" 라벨

## 12. 보호지침 학습 진행률 + 상황별 빠른 선택 (가장 큰 변경)

- `lib/protection-progress.ts` — localStorage 기반 9개 가이드 읽음 추적, `protection-progress-changed` 이벤트로 컴포넌트 동기화

- 9개 가이드 페이지 모두 마커 한 줄 삽입
- /protection 메인 상단 진행률 카드: "9개 중 N개 학습" + 그라디언트 진행 바 + 퍼센트
- 100% 달성 시 초록 그라디언트 + "🎉 모두 학습 완료"
- 각 가이드 카드 아이콘 우상단에 ✓ 초록 배지 (읽음 시)

### 12-2. 상황별 빠른 선택
헤더 바로 아래 "지금 어떤 상황인가요?" 가로 스크롤 칩 6개:
- 🚨 다친 아이 발견 → emergency
- 👶 새끼를 봤어요 → kitten
- 🍴 먹이 줘도 되나? → feeding
- ✂️ TNR 알아보기 → trapping
- 🏠 쉼터 만들기 → shelter
- 💊 약·영양제 → pharmacy

각 칩 컬러는 해당 가이드 카드와 통일.
