## §7. funnel 계측 부활기 — SQL 3왕복 끝에 서버 라우트 우회 (8067782)
- 발단: funnel_events 배포(7/18) 후 **0행**. QA·리서치가 REST 침투로 원인 특정 — anon INSERT가
  42501, 정책부 미실행. + 가드 선점 버그(실패해도 localStorage 가드 → 기기 영구 침묵).
- v1: 정책 재적용 SQL → 여전히 42501. v2: GRANT 추가 + 에디터 내 anon 역할 자가테스트 —
  **자가테스트의 rollback이 (에디터가 전체를 한 트랜잭션으로 묶는 탓에) 정책·GRANT까지 되돌리는
  자폭이었음**. v3: 테스트 블록 제거 → 그래도 42501 (원인 최종 미상 — 다른 프로젝트 실행 의심 포함).
- **종결**: RLS 의존 자체를 제거 — `/api/funnel` 서버 라우트(service role + step 화이트리스트 +
  anon_id 길이 검증 + IP 분당 10회 레이트리밋) 신설, funnel-repo가 이 라우트 호출(가드는 성공 후
  설정, 키 v2). 프로덕션 검증: POST → ok:true → DB 행 확인 → 테스트 행 삭제. **역사상 첫 성공 인서트.**
- 교훈 2개: ①Supabase SQL Editor에서 begin/rollback 자가테스트 금지(전체 트랜잭션 롤백)
  ②무음 실패(silent catch) 계측은 "배포 직후 프로덕션 실측 1회"가 완료 조건.
