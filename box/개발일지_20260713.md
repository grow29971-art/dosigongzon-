## 오늘 한 것

### 1. 크론 GET 이슈 최종 확인 (이상 없음)
- 크론 라우트 20개 전수 조사 — 전부 `export const GET = POST`(또는 별도 GET) 존재
- 프로덕션 무인증 호출 시 GET/POST 모두 401 (라우팅 정상, 405 아님)
- 결제 대사 크론 불일치 시 관리자 DM이 오게 돼 있으므로 조용하면 정상

### 2. 지도 관리자 버튼 미표시 — 원인 규명 + 수정 (d5f80d6, 배포)
- 데이터는 전부 정상: RLS(`admins_read_self`), admins 행
  (grow29971 = google/kakao/apple 통합 1계정, rias21125).
  ekgo9428@naver.com은 사용자 계정 아님(일반 유저) — 계정 원인 배제
- **실제 결함**: 관리자 체크가 첫 페인트 직후 idle 콜백에서 **1회만** 실행 —
  세션 복원이 늦거나 그 순간 네트워크가 튀면 isAdmin=false로 영구 고정, 재시도 없음
- **수정**: useAuth(onAuthStateChange 구독)의 user.id 변화에 반응하는 effect로 이동.
  세션이 늦게 복원돼도 그 시점에 재확인, 로그아웃 시 false 복귀

### 3. 홈 히어로 퀵 돌봄 확장 (00870ae, 배포)
- 밥주기 옆 `+` 버튼 → 카드 내 오버레이로 물/간식/건강/쉼터 1탭 기록
- 메모·사진이 필요한 기록은 "메모·사진과 함께 기록 →"으로 고양이 상세 연결
- **버그 수정**: "오늘 밥" 뱃지가 care_type 구분 없이 아무 돌봄 기록에나 켜지던 것
  → feed 유형만 집계하도록 수정 (헤더의 n/m 카운트도 동일)

### 4. 지도 마커 배회 애니메이션 — 위치 보호 2차 레이어 (39067d4, 배포)
- 사용자 요청: 등록 고양이가 실제 활동하듯 반경 5km를 불규칙하게 이동
- `roamCoord()` (cats-repo): id 해시 위상 + 서로 소 주기 3개(≈55/13/4분) 사인 합성
  → 결정적 배회 궤적. Date.now() 기반이라 서버 없이 모든 접속자가 같은 위치를 봄
- 마커(개별/동 클러스터) 250ms마다 갱신 — 평균 시속 ~54km, 틱당 ~4m (부드러움)
- 시뮬레이션 검증: 최대 반경 4.5km(√2 정규화로 5km 이내 보장)
- 보안 구조 정리: ① 등록 시 ±444m 오프셋(진짜 위치는 DB에도 없음)
  ② 비로그인 +500m 퍼징 ③ 화면 배회 5km — 화면 관찰로 급식소 특정 불가
- 반경은 `CAT_ROAM_RADIUS_M` 상수 하나로 조절 (5km면 동 라벨과 마커 위치가
  어긋나 보일 수 있음 — 어색하면 500m~1km로 낮추면 됨)
