## §1. 주간 출석 포인트 절반 하향 (커밋 a8ca46a)
쇼핑몰 1P=1원 실사용 대비 부채 축소. **실돈 포인트의 적립 경로는 주간 출석 마일스톤 하나뿐**임을
확인(돌봄일지·배틀은 게임 코인이라 무관 — 사장님 우려와 실제 구조가 달랐음).
- 3일 100P→**50P** / 5일 200P→**100P** / 7일 300P→**150P** (주 최대 600→300P, 월 ~1,300원/유저).
- 수정: `api/points/claim-weekly` + `WeeklyCheckinCard` 두 곳(값 반드시 동기 유지).
- 이미 수령분은 reason 유니크로 유지, 이중 지급 없음. DB 마이그레이션 불필요(금액이 코드에만).

## §2. 포인트 사용 한도 — 주문 금액의 30% (커밋 ff06ccd)
- **`lib/points-config.ts` 신설**: `POINTS_MAX_USE_RATE=0.3` + `maxPointsUsable()` 단일 소스.
  30% 상한과 기존 "최종 결제액 100원 이상" 중 낮은 쪽이 한도.
- 3중 적용: 체크아웃 UI(입력·최대사용 버튼·안내문) → 주문 생성(order-repo) → **결제 승인 서버 검증
  (payment/confirm, 최종 관문)**. UI 우회해 orders를 조작해도 승인 전 취소되고 청구 없음.
- 조정은 `POINTS_MAX_USE_RATE` 한 줄.

## §3. 포인트 실태 조회 — 부채 0 확인

checkin_days 전체 누적 8행(7/13 출시 후 3일 출석 도달자 없음). → §1·§2 하향이 **손해 보는 유저
0명인 타이밍에 적용됨.** 기존 유저 보상·공지 불필요.

## §4. 코인 지급량 전체 하향 (커밋 0553b48)
일일 확정 수입 75코인(출석50+로그인15+돌봄10)이 아이템 가격대(30~140) 대비 과함.
- 출석체크 50→**25** / 로그인 15→**8** / 돌봄일지 건당 2→**1**(일 최대 5). 일일 확정 75→38.
- 배틀 승 3→2·무 2→1·**패 1→0**, 보스 승 8→4·무 3→2·**패 1→0** (패배 무한 파밍 구멍 차단),
  주간 랭킹 200~20→100~10. ※§6에서 배틀 삭제로 이 항목들은 당일 소멸.
- UI는 서버 응답 값 표시라 하드코딩 수정 없음. "코인 +0" 표시만 조건부 숨김.

## §5. "PVP 없앤 거 아님?" — 실태 정정
내가 코드만 보고 "PVP 살아있다" 답했다가 정정. **실사이트 기준으론 카드게임 전체가 접근 불가**였음:
- 하단 탭 카드게임 7/10 숨김(BottomNav 주석), 마이페이지 진입 7/15 `SHOW_CARD_GAME=false`.
- 유일한 노출 잔재: 고양이 등록 축하 화면의 "배틀해보기" 링크 + 직접 URL.
- 교훈: 기능 상태 답변은 코드 존재가 아니라 **유저 도달 경로 기준**으로. (PVE 안내 문구
  "져도 코인" 잔재도 이때 수정 — 커밋 8e8a723)
