## 한 일

### 1. Meta Pixel CAPI 서버사이드 전환 추적 (feffe2f, 712c8d7)
**Why:** 쿠키 동의 거부·iOS ATT·광고 차단기 환경에서는 클라이언트 fbq가 비어있어 광고 conversion이 잡히지 않음. 광고 캠페인 효율이 실제보다 낮게 측정되는 문제.

**구현:**
- `lib/meta-capi.ts` — `sendMetaCAPIEvent(opts)` 서버 함수
  - SHA-256 해시 처리: `em` (이메일 소문자 trim 후), `external_id` (user.id)
  - `client_user_agent`, `client_ip_address`, `_fbp`/`_fbc` 쿠키 포함
  - `AbortSignal.timeout(5000)` — 응답 지연 방어
  - `META_PIXEL_ACCESS_TOKEN` 미설정 시 silent skip → 점진 도입 가능
- `/api/auth/callback`에서 **첫 가입자 전체(소셜+이메일)** CompleteRegistration 발사
  - 처음에는 firstSocialSignup만 발사했다가 이메일 가입자 누락 발견 → firstSignup으로 확장
  - 이메일 가입자는 welcome 페이지 안 거쳐서 client fbq도 발사 안 되던 누락 동시 해결
- `event_id = user.id`로 client·server dedup — 양쪽 발사돼도 한 번만 카운트
- `lib/meta-pixel.ts` `trackPixelOnce`에 eventID 옵션 추가, welcome 페이지에서 user.id 전달

**활성 조건:** `NEXT_PUBLIC_META_PIXEL_ID` (이미 라이브 1611645713230757) + `META_PIXEL_ACCESS_TOKEN` (Meta Business Manager → Conversions API → System User Access Token). 토큰만 Vercel env에 추가하면 자동 활성.

메모리 `project_dosigongzon_meta_capi.md` 신규.

### 2. admin/inbox `hospital_closed` 처리 핸들러 (df7eef8)
**버그:** 5/19 작업으로 병원 폐업 신고가 reports 테이블에 3회 누적 후 자동 hidden 처리되도록 바뀌었는데, `ReportTargetType`에 `hospital_closed`가 추가 안 됐고 admin inbox UI는 처리 분기가 없었음. 결과:
- "대상 삭제" 클릭 시 잘못된 라벨 "이 고양이 등록을(를) 정말 삭제할까요?" 표시
- 실제 API 호출은 안 됨 (if 분기 모두 miss)
- 신고 status만 silently resolved로 마킹 → admin이 처리한 줄 알지만 병원은 그대로 보임

**수정:**
- `ReportTargetType`에 `hospital_closed` 추가
- `hideHospitalByAdmin` / `restoreHospitalByAdmin` repo 함수 신설 (`rescue_hospitals.hidden` 토글 — 삭제 아닌 토글이라 오신고 복원 가능)
- inbox 액션 라벨 분기: `hospital_closed`면 "병원 숨김" + "병원 복원", 아니면 기존 "대상 삭제" + "신고자 정지"
- 같은 hospitalId의 pending 신고 일괄 status 마킹

### 3. service-role cats 조회 `hidden=false` 필터 누락 보정 (9b4ed59)
**보안/UX 약점:** service_role 키로 createClient한 API/cron들이 RLS를 우회하므로 신고로 hidden 처리된 cats가 통계·푸시에 그대로 잡히던 문제. 3 군데 발견:
- `/api/social-proof`: 비로그인 홈에 노출되는 `newCatsThisWeek`, `totalCats`, `rescueCount`가 부풀려져 표시
- `/api/cron/engagement-push`: 주간 푸시 메시지의 "새 친구 N마리"가 hidden 포함
- `/api/cron/health-alert-push`: 긴급 알림 대상 cats에 hidden 포함 → 신고당한 cat에 대해서도 알림 발송 가능

세 곳 모두 `.eq("hidden", false)` 추가.

`weekly-postcard-push`는 본인이 등록한 cat 단순 lookup이라 영향 적어 패스.
