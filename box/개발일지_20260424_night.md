## 2. 첫 시도 실패 — Client Secret 불일치 (디버깅)

### 2-1. 증상
배포 후 본인 휴대폰으로 카카오 로그인 시도:
```
Unable to exchange external code: QVnQ
코드: unexpected_failure
```

### 2-2. 원인 추적
- Supabase가 카카오로 code → access_token 교환 단계에서 실패
- Supabase Dashboard에 입력된 Client Secret이 카카오 콘솔의 실제 값과 다름
- 정황상 과거 dev 단계에 잘못된 값 입력된 채로 방치되어 있었음

### 2-3. 카카오 콘솔에서 진짜 Client Secret 찾기 (콘솔 신버전 미로)

신버전 카카오 콘솔에서 Client Secret 위치를 찾는 데 시간 걸림:
- "카카오 로그인 → 고급" 페이지: **로그아웃** 리다이렉트 URI만 있음 (X)
- "앱 설정 → 고급" 페이지: 빈 페이지 (X)
- **"앱 설정 → 플랫폼 키" 페이지**: REST API 키 카드 안에 **"클라이언트 시크릿"** 태그
  → 클릭하면 Client Secret 페이지 (✅)

Client Secret 페이지에서:
- 카카오 로그인용 코드: `17GBfH...` (32자리)
- 활성화 ON 상태 (REST API 키 발급 시 자동 활성)

이 값을 Supabase에 정확히 입력 → 재시도 → **성공** ✅
