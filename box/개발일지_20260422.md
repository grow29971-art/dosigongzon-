## 1. 타이틀 부여 버그 fix (`15f16a6`)

증상: `/admin/users`에서 유저에게 타이틀 부여 시 `"admin_title은 관리자만 변경 가능합니다."` 에러.

원인:
- 어제 추가한 `guard_profile_sensitive_columns()` 트리거가 `auth.uid()`로 admin 판정
- `/api/admin/set-title`은 `service_role` 키로 호출 → 세션 내 `auth.uid()`가 NULL
- 서버 측에서 이미 admin 검증 마쳤지만 트리거는 그걸 모름 → 차단

해결 (`box/supabase_profiles_guard_fix_service_role.sql`):
- `auth.role() = 'service_role'`이면 id 변경만 차단하고 통과
- 일반 유저 경로는 기존 검증 유지 (admin_title / invite_code / invited_by / id)
