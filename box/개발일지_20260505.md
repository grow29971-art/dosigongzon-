## 4. 자동재생/자동 다음 영상/음소거 (UX 작업이 가장 어려웠음)

### 4-1. 자동재생 + 음소거 토글 + 자동 다음 영상
- 진입 즉시 mute=1로 자동재생 (브라우저 정책).
- iframe `infoDelivery` postMessage에서 `playerState=0` (ENDED) 감지 → 다음 카드로 `scrollIntoView`.
- 우상단 음소거 토글 + 첫 인터랙션 시 자동 unmute (TikTok 패턴).

### 4-2. 버그: 모든 iframe 동시에 소리 남
**원인**: embed URL `autoplay=1` → 30개 iframe 한꺼번에 자동재생 → unmute 시 동시 출력.
**해결**: `autoplay=1` 제거. iframe `onLoad` + IO에서 보이는 카드만 `playVideo` postMessage.

### 4-3. 버그: 스와이프 시 소리 났다 안 났다
**원인**: lazy-load되는 iframe은 항상 `mute=1` URL로 시작 → 사용자가 이미 unmute 했어도 새 iframe은 음소거 상태 그대로.
**해결**: `mutedRef`로 최신 muted 상태 추적 → iframe `onLoad` 시점에 현재 상태 적용.

### 4-4. 버그: 음소거 버튼 단일 탭 안 먹힘 (꾹 눌러야 됨)
**원인**: 자동 unmute 리스너가 mute 버튼 탭에도 발동 → `setMuted(false)` 후 버튼 onClick `setMuted(m=>!m)` → React 함수형 setState가 stale 상태 잡아 toggle 상쇄.
**해결**: 버튼에 `data-mute-toggle` 마커 + 자동 unmute 리스너에서 `target.closest("[data-mute-toggle]")` 체크해 제외.

### 4-5. 힌트 메시지 수정
원본: "🔇 화면을 한 번 탭하면 소리가 켜져요"
→ YouTube iframe 내부 탭은 cross-origin이라 부모 페이지에 이벤트 전달 안 됨. "탭"이 무의미.
**해결**: "🔇 우상단 버튼으로 소리 켜기"로 변경 + top:64 → 108로 내려서 상단 바와 안 겹치게.

## 6. YouTube 자동 임포트 (가장 큰 기능)
관리자가 일일이 등록 안 해도 매일 새 길고양이 shorts 자동 수집.

- `lib/youtube-api.ts`: `searchShorts(query, max, order)` + `getVideoStats(ids)`.
- `/api/cron/import-shorts`: Vercel cron 매일 00:00 UTC (= 09:00 KST) + 어드민 수동 호출 가능.
- 인증: `CRON_SECRET` (cron) 또는 admin 쿠키 (수동) 둘 다 허용.

### 6-1. 검색어 풀 진화
- 시작: 4개 (`고양이 shorts` 등) → 매번 같은 결과 → 0개 추가.
- 확장: 약 60개 (한국어 일반/행동/품종/감정 + 영어 동등). 매 호출마다 `pickRandom(POOL, N)` + 정렬도 랜덤(`date/viewCount/relevance`).

### 6-2. 조회수 필터
- search.list엔 조회수 없음 → `videos.list?part=statistics`로 50개씩 일괄 조회 후 `MIN_VIEW_COUNT >= 10,000` 필터링.

### 6-3. 쿼터 관리
무료 쿼터 10K units/일. search.list = 100 units/호출.
- 처음: 5 queries × 100 = 500/호출 → 20회/일 한도 → 테스트 중 초과.
- 조정: `QUERIES_PER_CALL=3`, `PER_QUERY_LIMIT=30` → 300/호출 → 33회/일.
- 쿼터 초과 시 어드민 페이지에 명확한 안내 (한국시간 17:00 리셋).

### 6-4. 환경변수 등록 시 함정
PowerShell에서 `vercel env add` 파이프로 값 전달했더니 빈 문자열로 저장됨 (`echo "..." | ...` 도, `--value` 플래그도). bash로도 동일. 결국 Vercel 대시보드에서 수동 입력으로 해결. CLI 버전(51.6.1) 버그 의심.

## 가져갈 교훈
- **lazy iframe + 상태 동기화**: lazy-load한 iframe은 마운트 시점의 props를 못 받음. ref로 최신 상태 노출 필수.
- **IO 시점 vs iframe 로드 시점 race**: 둘 다에서 트리거하되 서로의 ready 상태 ref로 가드.
- **postMessage with `enablejsapi=1`**: YouTube IFrame API 안 불러와도 `infoDelivery` 이벤트 받을 수 있음 — `listening` 메시지로 구독만 하면 됨.
- **CSP frame-src는 명시 필수**: `default-src` fallback 안 받음. 새 임베드 도메인 추가 시 항상 frame-src에 명시.
- **Vercel CLI 환경변수 stdin 버그**: 51.6.1에서 파이프·--value 모두 빈 값 저장. 대시보드 직접 입력 권장.
