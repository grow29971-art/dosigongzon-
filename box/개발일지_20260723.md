### 확정 위험
- 🔴 **B. profiles authenticated PII 대량조회 (HIGH·진짜 위험):** profiles SELECT가 `USING(true)`인데 `authenticated` 롤에 `invite_code/coins/marketing_push_enabled/invited_by` 컬럼 grant가 열려 있음. 과거 email 핫픽스가 anon만 막고 authenticated는 방치. **잠금 SQL은 주석 계획만 있고 파일 자체가 box/에 없음**(`supabase_profiles_authenticated_lockdown_migration.sql` 부재 확인). 악마의 변호인이 "셋 중 가장 확실"로 확정. 익스플로잇: 아무 계정 로그인 → `GET /rest/v1/profiles?select=invite_code,coins,marketing_push_enabled` → 회원 ~313명 덤프. 앱 UI로는 재현 불가, JWT로 REST 직접 치면 뚫림. 통신판매업·개인정보 감사 맥락 지적 소지.

### 조건부 (라이브 검증 필요)
- 🟠 **A. 무한 코인 RPC (조건부):** `buy_shop_item_atomic(p_user_id,p_item_key,p_price)`가 security definer인데 p_price 음수/auth.uid() 미검증 → 원본 그대로면 무한 발행+IDOR. **하지만** 수정 SQL(`security_audit_20260716_2`)이 authenticated execute를 정확히 revoke하고, 앱 라우트는 가격 서버 고정+service_role 호출이라 정상 경로엔 구멍 없음. 문제: 앱이 service_role로 돌아 **revoke 적용 여부와 무관하게 앱은 동작** → 코드로는 라이브 상태 판정 불가. MEMORY "SQL 6종 실행이 보안 마무리 조건"과 일치. 검증법: 라이브에서 유저 JWT+anon 키로 `rpc/buy_shop_item_atomic {p_price:-1}` → 403 확인.
