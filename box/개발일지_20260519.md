### 1. 서클 가시성·활용 동기 4중 강화 (4ca47d7)
**의도**: Private Circle 백엔드는 탄탄한데 일상 동선에서 가시성·활용 동기 부족. 4개 레이어로 노출 강화.

**A. 지도 핀에 visibility 배지** (`map/page.tsx`)
- selectedCat 카드 헤더에 **본인 핀일 때만** 컬러 배지 표시 (🤝 내 서클 / 🔒 나만 보기)
- public은 미표시 — 기본값이라 노이즈

**B. 마이페이지·홈 서클 카운트 노출**
- `lib/circles-repo.ts`: `countMyAcceptedCircleMembers()` 추가 (서클 없으면 자동 생성 안 하고 0 반환)
- `/mypage` 진입 링크에 **"내 서클 (N)"** 멤버 수 뱃지 + 동적 카피
  - 0명: "믿는 이웃을 초대해 안전한 돌봄 시작하기"
  - N명: "N명의 이웃과 함께 안전하게 돌봐요"
- `HomeAuthed`도 동일 카운트 로드

**C. 온보딩 카드 4단계 — "믿는 이웃 초대"**
- `OnboardingCard`: `hasCircleMember` prop + ShieldCheck 아이콘 step 4 추가 (선택)
- 부제 "3단계만…" → "마지막은 선택 — 믿는 이웃과 안전한 돌봄까지"

**D. 카카오톡 초대 링크**
- `box/supabase_circle_join_rpc_migration.sql`: **`join_circle_by_owner(uuid)` SECURITY DEFINER RPC**
  - `auth.uid()` 본인 검증, owner의 서클이 없으면 자동 생성, pending → accepted 승격
  - 기존 `circle_members_invite` RLS는 owner만 insert 허용 → 링크 자기 가입 불가 문제 우회
- `/circle/join/[ownerId]` 페이지: 로그인 가드, owner 프로필 카드, 수락/거절 버튼, 결과 화면 (accepted/already/rejected/self)
- `/mypage/circle`: 옐로 카드로 **카카오톡 공유** + **링크 복사** 버튼 추가 (`shareToKakao` 사용)

### 3. private/circle 핀도 통계 카운트에 포함 (0de2d1e)
**요구**: "나만 보기" 핀도 전체 통계에는 잡혀야 함. 위치·사진·이름 같은 식별 정보는 그대로 비공개. **카운트만 가산**.

**`box/supabase_cat_total_count_rpc.sql`**:
- `total_cat_count()` — visibility 무관 전체 카운트
- `total_danger_cat_count()` — 위험 상황 카운트
- `recent_cat_count_by_region(days)` — 동네별 최근 N일 + 활동 케어테이커
- 모두 `SECURITY DEFINER stable`로 RLS 우회, `anon`·`authenticated` grant

**코드 교체**:
- `HomeLanding.getLandingData`: 전국 catCount → `total_cat_count` RPC
- `about/page.getStats`: cats → `total_cat_count` RPC
- `opengraph-image.getStats`: cats·urgent → `total_cat_count`·`total_danger_cat_count` RPC
- `region-activity.fetchActiveRegions`: 최근 cats select → `recent_cat_count_by_region` RPC (활발한 동네 TOP 3에 private 포함)

식별 정보(cats select 개별 row)는 `cats_read_by_visibility` RLS로 여전히 public·본인·서클 멤버에게만 노출.
